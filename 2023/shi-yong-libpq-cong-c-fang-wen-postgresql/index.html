使用libpq从C访问PostgreSQL

postgresql
数据库
发布日期:   2023-03-21

如果我们想完全控制我们的客户端应用程序，我们需要可以考虑创建自定义接口。这就是libpq的作用。

libpq库提供的函数分为三类。

  • 数据库连接和连接管理

  • SQL语句执行;

  • 检索查询结果集

就像许多产品在许多版本中成长和发展一样，经常有在libpq中执行相同操作的方法不止一种。在本章中，我们将专注于最常见的方法，并提供有关任何替代方案和实例的提示它们可能特别适用。

使用libpq库

所有使用libpq库的PostgreSQL客户端应用程序都必须这样编写源代码，代码包括定义libpq提供的函数的适当头文件和应用程序必须连接到正确的库，其中包含这些函数的代码。

客户端应用程序被称为PostgreSQL的前端程序，必须包含头文件libpq-fe.h (fe用于前端)。此头文件提供了libpq函数和隐藏PostgreSQL的内部工作方式。坚持使用libpq-fe.h将确保程序能够在未来的版本中编译libpq。头文件安装在PostgreSQL安装的include子目录中(在UNIX和Linux上，默认是/usr/local/pgsql/include)。我们需要引导C编译器到这个目录，以便它可以使用-L选项找到头文件。

PostgreSQL发布版提供的头文件libpq-int.h包含libpq使用的内部结构的定义，但不建议在普通客户端中使用它

libpq库将安装在PostgreSQL安装的lib目录下默认为/usr/local/pgsql/lib)。为了将libpq函数合并到应用程序中，我们需要

连接到该库。最简单的方法是告诉编译器使用-lpq进行连接，并使用-L选项指定PostgreSQL库目录作为查找库的地方。

一个典型的libpq程序具有以下结构:

#include <libpq-fe.h>
main()
{
 /* Connect to a PostgreSQL database */
 LOOP:
 /* Execute SQL statement */
 /* Read query results */
 
 /* Disconnect from database */
}

程序将通过使用类似这样的命令行:

gcc -o program program.c -I /usr/local/pgsql/include -L /usr/local/pgsql/lib -lpq

如果您使用的是Linux发行版(如Red)中的PostgreSQL安装在Linux上，您可能会发现libpq库安装在编译器搜索的默认位置，所以只需要指定include directory选项，如下所示:

$ gcc -o program program.c -I /usr/local/pgsql/include -lpq

其他Linux发行版和其他平台安装可能会放置包含文件和不同地libpq。通常，它们将位于基础目录的include和lib目录中d额PostgreSQL安装目录。

建立数据库连接

一般来说，PostgreSQL客户端应用程序在运行时可以连接到一个或多个数据库。事实上，我们甚至可以在同样的时间连接到由许多不同服务器管理的许多数据库。libpq库提供了创建和维护这些连接的函数。

当我们连接到服务器上的PostgreSQL数据库时，libpq会返回一个句柄数据库连接。这表示为头文件中定义的一个内部结构 PGconn，我们可以将其视为类似于文件句柄。许多libpq函数需要 PGconn 指针参数用于标识目标数据库连接，方式与此类似C语言中的标准I/O库使用文件指针。

创建一个新的数据库连接

使用PQconnectdb创建一个新的数据库连接，如下所示：

PGconn *PQconnectdb(const char *conninfo);

PQconnectdb函数返回一个指向新连接描述符的指针。返回如果无法分配新的描述符，可能是因为缺少描述符，result将为NULL

分配新的描述符所需的内存。从PQconnectdb返回的非空指针可以做到这一点。然而，这并不意味着连接成功。我们得检查一下连接器的状态。

PQconnectdb的单个参数是一个字符串，指定要连接到哪个数据库。其中嵌入了各种选项，我们可以使用它们来修改连接的方式。

Conninfo 字符串参数由空格分隔的选项组成，形式为option=value。表1列出了最常用的选项及其含义。该表格还显示未指定连接选项时默认使用的环境变量。

表1 常见的PQconnectdb连接选项

选项 含义 默认环境变量
host 要连接的服务器的名称 $PGHOST或localhost（如果没有设置）
hostaddr 要连接的服务器IP地址 $PGHOSTADDR
dbname 连接的数据库 $PGDATABASE或用户名（如果没有设置）
port 服务器上要连接的TCP/IP端口 $PGPORT或5432（如果没有设置）
user 连接时使用的用户名 $PGUSER或用户名（如果没有设置）
password 指定用户的密码 $PGPASSWORD或none（如果没有设置）

例如，要连接到本地机器上的db数据库，我们将使用Conninfo字符串，像这样:

"dbname=db"

若要在选项值中包含空格，或输入空值，该值必须加引号使用单引号，如下所示:

"host=db password='123456' user=postgres"

host选项指定要连接的服务器。PQconnectdb调用将导致查找名称以确定服务器的IP地址，以便建立连接。

通常，这是通过使用域名服务(DNS)完成的，可能需要一段时间来完成。如果你已经知道服务器的IP地址，可以使用hostaddr选项

端口选项。

使用环境变量进行连接

这些选项也可以通过环境变量来指定，如表1所示。例如，如果 conninfo 参数中没有设置 host 选项，那么 PQconnectdb 将进行查询

检查环境是否设置了变量 PGHOST。如果是，值 $PGHOST 将被用作要连接的主机名。我们可以编写一个客户端程序，用一个空值调用PQconnectdb 字符串并通过环境变量提供所有选项:

#include <libpq-fe.h>

int main()
{
 PGconn *conn = PQconnectdb("");
 ...
}

We could then assign a few environment variables and execute the program like so:

$ PGHOST=beast PGUSER=neil ./program

检查连接状态

如前所述，PQconnectdb 返回一个非空的连接句柄并不会表示连接没有错误。我们需要使用另一个函数 PQstatus 来检查连接的状态：

ConnStatusType PQstatus(const PGconn *conn);

ConnStatusType 是一种枚举类型，其中包括一些常量 CONNECTION_OK 和 CONNECTION_BAD。PQconnectdb 将返回这两个值之一，具体取决于连接是否成功。对于其他连接方法使用 ConnStatusType 中的其他状态值，例如使用 PQconnectStart 进行异步连接。

关闭连接

当我们完成一个数据库连接时，我们必须关闭它，就像以前一样打开文件描述符。我们通过将连接描述符指针传递给 PQfinish 来实现:

void PQfinish(PGconn *conn);

对 PQfinish 的调用允许由libpq库释放连接。

重置连接

如果连接出现问题，尝试重置它可能是有用的。PQreset 函数为为此目的而设的。它将关闭到后端服务器的连接，并尝试创建一个新的连接，参数与原来的连接设置相同：

void PQreset(PGconn *conn);

编写连接程序

我们现在可以编写可能最短有用的 PostgreSQL 程序(connect.c)，它可以用于检查是否可以与特定数据库建立连接。我们将使用环境变量来将选项传递给 PQconnectdb，我们可以考虑使用命令行参数甚至硬编码，如果它适合我们的应用程序。

#include <stdlib.h>
#include <libpq-fe.h>

int main()
{
    PGconn *myconnection = PQconnectdb("");
    if(PQstatus(myconnection) == CONNECTION_OK)
        printf("connection made\n");
    else
        printf("connection failed\n");
    PQfinish(myconnection);
    return EXIT_SUCCESS;
}

让我们来执行这个程序，首先不带必需的选项，然后在分配 PGDATABASE 和 PGUSER 环境变量:

$ gcc -o connect -I /home/postgres/pg96/include connect.c -L /home/postgres/pg96/lib -lpq
$ ./connect
connection made

使用Makefile

在前面的例子中，我们对编译器使用了适当的 -L 和 -I 选项。这些总是需要使用libpq编译程序。如果我们使用makefile来控制编译，我们可以将这些选项分别添加到 CFLAGS 和 LDLIBS 变量中，这简化了编译。

下面是一个非常简单的makefile，可用于编译其中的所有示例程序。

# 示例程序的Makefile
# 在PostgreSQL开始时
# 编辑基本目录
# PostgreSQL安装
INC=/home/postgres/pg96/include
LIB=/home/postgres/pg96/lib
CFLAGS=-I$(INC)
LDLIBS=-L$(LIB) -lpq
ALL=async1 async2 connect create cursor cursor2 print select1 select2 import
all: $(ALL)
clean : 
 @rm -f *.o *~ $(ALL)

现在我们可以通过简单地运行make(作为所有的程序)一次构建所有的程序，在makefile:all中作为第一个目标的依赖项指定)。我们可以建一个使用命令 make * Program 编程(其中 Program 是我们想要的程序的名称构建)。

检索连接错误的信息

注意，PQstatus 和 PQfinish 都可以处理连接描述符为NULL的指针，因此，在我们的示例中，我们之前没有检查 PQconnectdb 的返回结果是否有效。调用 PQstatus 和 PQfinish。可以获取可读字符串连接的状态或通过调用 PQerrorMessage 描述发生的错误：

char *PQerrorMessage(const PGconn *conn);

这个函数返回一个指向描述性字符串的指针。这个字符串将被其他libpq函数覆盖，因此应该在调用 PQerrorMessage 后立即使用或在调用其他libpq函数之前复制。

例如，我们可以让连接失败消息更有帮助，如下所示:

printf("connection failed: %s", PQerrorMessage(myconnection));

然后我们会看到如下信息更丰富的错误消息：

connection failed: FATAL: database “neil” does not exist

学习连接参数

如果我们在建立连接之后需要更多关于连接的信息，我们可能会考虑直接使用 PGconn 结构的成员(定义在 libpq-fe.h 中)，但这将是

这是个坏主意。这是因为代码可能在libpq的未来版本中出现错误 PGconn 内部结构发生变化。尽管如此，我们可能确实需要了解更多

关于连接，因此libpq提供了许多访问函数，这些函数返回值连接的属性：

  • char *PQdb(const PGconn *conn): Returns the database name.
  • char *PQuser(const PGconn *conn): Returns the username.
  • char *PQpass(const PGconn *conn): Returns the user password.
  • char *PQhost(const PGconn *conn): Returns the server name.
  • char *PQport(const PGconn *conn): Returns the server port number.
  • char *PQoptions(const PGconn *conn): Returns the options associated with a connection.

所有这些值在连接的生命周期内都不会改变。

使用libpq执行SQL

由 PQexec 函数发起查询过程:

PGresult *PQexec(PGconn *conn, const char *sql_string);

将SQL语句传递给 PQexec，通过非null连接到服务器 Connection conn 执行它。结果通过 PGresult 结构表示。当没有数据返回时，PQexec 将返回一个指向result结构的有效非空指针，它不包含数据记录。

在极少数情况下，如果没有足够的内存分配新的result结构，PQexec 可能会返回NULL指针。

传递给 PQexec 的字符串可以包含任何有效的SQL语句，包括查询，插入、更新和数据库管理命令。它们相当于SQL语句使用PSQL命令行工具运行，只是我们不需要在字符串中以分号结尾来标记语句的结束。下面是稍后会用到的一些例子:

PQexec(myconnection, "SELECT customer_id FROM customer");
PQexec(myconnection, "CREATE TABLE number (value INTEGER, name VARCHAR)");
PQexec(myconnection, "INSERT INTO number VALUES (42, 'The Answer')");

注意，SQL语句中的任何双引号都需要用反斜杠进行转义，这在psql中是必要的。

与连接结构一样，result对象在使用完毕后也必须被释放，可以使用 PQclear 来实现这一点，它也可以处理NULL指针。请注意，即使连接关闭，结果也不会自动清除，因此可以在需要时无限期保存：

void PQclear(PGresult *result);

确定查询状态

我们可以使用 PQresultStatus 函数探测SQL语句的执行状态，该函数返回构成枚举类型 ExecStatusType 的多个值中的一个:

ExecStatusType PQresultStatus(const PGresult *result);

表2列出了最常见的状态类型。其他状态类型表示服务器出现了一些意想不到的问题，例如正在备份或脱机。

表2 常见PQresultStatus状态类型

状态类型 说明
PGRES_EMPTY_QUERY 不需要数据库访问;通常是一个空查询字符串的结果。这种状态通常表明客户端程序有问题，发送的查询不需要服务器做任何工作。
PGRES_COMMAND_OK 成功;命令不返回数据。这种状态意味着SQL执行正确，而且语句的类型是不返回数据的，例如CREATE TABLE。
PGRES_TUPLES_OK 成功;查询返回零行或多行。这种状态意味着SQL执行正确，而且语句的类型可能返回数据，例如SELECT。在本例中，这并不意味着有数据可以返回。需要进一步调查，以确定实际有多少数据可用。
PGRES_BAD_RESPONSE 失败；服务器响应不清楚。这表明SQL执行失败了。
PGRES_NONFATAL_ERROR 失败；非致命的，可以重试。这表明SQL执行失败了。
PGRES_FATAL_ERROR 失败；致命的，不能重试。这表明SQL执行失败了。

下面是一个代码片段的例子，它使用 PQresultStatus 来确定调用 PQexec 的精确结果:

PGresult *result;
result = PQexec(myconnection, "SELECT customer_id FROM customer");
switch(PQresultStatus(result)) {
    case PGRES_TUPLES_OK:
     /* may have some data to process, find out */
     if(PQntuples(result)) {
         /* process data */
         break;
     }
     /* no data, drop through to no data case */
    case PGRES_COMMAND_OK:
         /* all OK, no data to process */
         break;
    case PGRES_EMPTY_QUERY:
         /* server had nothing to do, a bug maybe? */
         break;
    case PGRES_NONFATAL_ERROR:
         /* can continue, possibly retry the command */
         break;
    case PGRES_BAD_RESPONSE:
    case PGRES_FATAL_ERROR:
    default:
     /* fatal or unknown error, cannot continue */
}

在本章后面的“从查询结果中提取数据”一节中，当我们返回到 SELECT 的 PGRES_TUPLES_OK 案例时，将更详细地介绍 PQntuples。

PQresStatus 是一个有用的函数，可以帮助解决故障。这个函数将结果状态码转换为可读字符串:

const char *PQresStatus(ExecStatusType status);

当发生错误时，我们可以通过调用 PQresultErrorMessage，和连接时调用的方式差不多：

const char *PQresultErrorMessage(const PGresult *result);

使用PQexec执行查询

让我们看一些执行SQL语句的简单例子。在数据库中使用一个非常小的表作为尝试。稍后，将在示例客户表上执行一些操作，以返回大量数据。

创建一个名为number的数据库表，它存储数字和英文描述。表中的条目如下所示:

 value | name
-------+-------------
 42 | The Answer
 29 | My Age
 29 | Anniversary
 66 | Clickety-Click

为了创建表并向表中插入值，我们只需要调用 PQexec，使用包含需要执行的SQL查询的适当字符串。程序将包含这样的调用:

PGconn *myconnection;
...
PQexec(myconnection,"CREATE TABLE number (value INTEGER, name VARCHAR)");
PQexec(myconnection,"INSERT INTO number VALUES (42, 'The Answer')");

我们需要处理出现的错误。例如，如果表已经存在，我们尝试创建它时会得到错误。在创建已经存在的数字表时，PQresultErrorMessage 将返回这样一个字符串:

ERROR: Relation ‘number’ already exists

为了让事情变得更简单，开发一个自己的函数来执行SQL语句，检查结果并打印错误。

有了它，我们几乎可以像在psql中输入命令一样轻松地执行SQL查询。将这段代码保存在文件 create.c 中

#include<stdlib.h>
#include<libpq-fe.h>
void doSQL(PGconn *conn, char *command)
{
    PGresult *result;
    printf("%s\n", command);
    result = PQexec(conn, command);
    printf("status is %s\n", PQresStatus(PQresultStatus(result)));
    printf("result message: %s\n", PQresultErrorMessage(result));
    PQclear(result);
}

int main()
{
    PGresult *result;
    PGconn *conn;
     conn = PQconnectdb("");
     if(PQstatus(conn) == CONNECTION_OK) {
         printf("connection made\n");
         /* doSQL(conn, "DROP TABLE number"); */
         doSQL(conn, "CREATE TABLE number ( \
         value INTEGER, \
         name VARCHAR \
         )");
         doSQL(conn, "INSERT INTO number values(42, 'The Answer')");
         doSQL(conn, "INSERT INTO number values(29, 'My Age')");
         doSQL(conn, "INSERT INTO number values(29, 'Anniversary')");
         doSQL(conn, "INSERT INTO number values(66, 'Clickety-Click')");
     }
     else 
         printf("connection failed %s\n", PQerrorMessage(conn));
     PQfinish(conn);
     return EXIT_SUCCESS;
}

这里，我们创建了一个表，并向其中添加了一些元素。如果重新运行程序，会看到一个致命错误，因为我们无法第二次创建表。取消DROP TABLE命令的注释，将程序更改为每次运行时销毁并重新创建表的程序。

当然，在生产环境的代码中，我们不会如此轻率地对待错误。在这里，为了简洁起见，我们省略了从doSQL返回结果，不管失败与否，我们都继续执行。

编译和运行时，程序应该显示一些执行和状态信息:

$ make create
$ PGDATABASE=bpfinal ./create
connection made
...

INSERT INTO number VALUES(66, 'Clickety-Click')
status is PGRES_COMMAND_OK
result message:

创建一个变量查询

为了在SQL中包含用户指定的数据，可以创建一个字符串来传递给 PQexec，其中包含我们想要的值。要将所有个位数的整数相加，可以这样写：

for(n = 0; n < 10; n++) {
    sprintf(buffer,"INSERT INTO number VALUES(%d, 'single digit')", n);
    PQexec(buffer);
}

更新和删除行

如果想更新或删除表中的行，可以分别使用update和delete命令：

UPDATE number SET name = 'Zaphod' WHERE value = 42
DELETE FROM number WHERE value = 29

如果要在程序中添加对 PQexec(或doSQL)的适当调用，这些命令会首先将数字42的描述文本改为 Zaphod，然后将29的两个条目都删除。我们可以使用psql检查更改的结果:

$ psql -d bpfinal
bpfinal=# SELECT * FROM number;
 value | name
-------+-----------------
 66 | Clickety-Click
 42 | Zaphod
(2 rows)

删除和更新可能会影响表中的不止一行(或者PostgreSQL喜欢称之为元组);因此，知道有多少行被修改了通常是很有用的。我们可以通过调用 PQcmdTuples 来获取这些信息：

const char *PQcmdTuples(const PGresult *result);

奇怪的是，PQcmdTuples 返回的不是你预期的整数，而是包含数字的字符串。我们可以修改doSQL函数，简单地报告受影响的行:

printf("#rows affected %s\n", PQcmdTuples(result));

我们现在会看到，PQcmdTuples 返回一个空字符串，对于那些对行没有任何影响的命令，比如CREATE table，返回字符串”1”和”2”，对于那些对行有影响的命令，比如INSERT和DELETE。我们必须仔细区分真正不影响任何行的命令和那些失败因而不影响任何行的命令。我们必须始终检查结果状态以确定错误，而不仅仅是受影响的行数。

从查询结果中提取数据

到目前为止，我们只关注没有返回任何数据的SQL语句。现在来考虑如何处理 PQexec 调用返回的数据，也就是SELECT语句的结果。当使用 PQexec 执行SELECT时，结果集将包含查询返回的数据的信息。查询结果处理起来可能有点麻烦，因为我们并不总是确切地知道预期会发生什么。如果执行SELECT，我们事先不知道返回的是0、1还是数百万行。如果在SELECT查询中使用通配符(*)，我们甚至不知道将返回哪些列，以及它们的名称是什么。一般来说，我们希望应用程序只选择指定的列。这样，如果数据库的设计发生了变化，比如添加新列时，不依赖于新列的函数仍然可以正常工作。有时，如果我们正在编写一个通用的SQL程序，它如果能接受以通用的方式编程就更好了，使用libpq，我们可以做到。还有几个函数需要学习：

当 PQexec 执行SELECT语句没有错误时，我们希望看到结果的状态 PGRES_TUPLES_OK。下一步是确定结果中有多少行集。我们通过调用 PQntuples 来获取结果中的总行数可以是零

int PQntuples(const PGresult *result);

可以通过调用 PQnfields 获取元组中字段(属性或列)的数量

int PQnfields(const PGresult *result);

结果中的字段从0开始编号，可以通过 PQfname 取得字段的名称

char *PQfname(const PGresult *result, int index);

字段的长度由 PQfsize 给出

int PQfsize(const PGresult *result, int index);

对于固定大小的字段，PQfsize 返回特定列中某个值占用的字节数。对于变长字段，PQfsize 返回-1。

给定名称的列的索引号可以通过调用 PQfnumber 取得:

int PQfnumber(const PGresult *result, const char *field);

让我们修改一下doSQL函数，打印出一些关于SELECT查询返回数据的信息。这是我们的下一个版本:

void doSQL(PGconn *conn, char *command);
{
     PGresult *result;
     printf("%s\n", command);
     result = PQexec(conn, command);
     printf("status is %s\n", PQresStatus(PQresultStatus(result)));
     printf("#rows affected %s\n", PQcmdTuples(result));
     printf("result message: %s\n", PQresultErrorMessage(result));
     switch(PQresultStatus(result)) {
        case PGRES_TUPLES_OK:
        {
            int n = 0;
            int nrows = PQntuples(result);
            int nfields = PQnfields(result);
            printf("number of rows returned = %d\n", nrows);
            printf("number of fields returned = %d\n", nfields);
            /* Print the field names */
            for(n = 0; n < nfields; n++) {
                printf(" %s:%d", PQfname(result, n), PQfsize(result, n));
            }
            printf("\n");
        }
     }
     PQclear(result);
}

现在，当执行SELECT时，可以看到返回数据的特征：

doSQL(conn, "SELECT * FROM number WHERE value = 29");

这个调用的输出如下：

注意，PQcmdTuples 对不能影响行的查询返回一个空字符串，而 PQresultErrorMessage 在没有错误的情况下返回一个空字符串。现在我们可以从结果集中每行返回的字段中提取数据了，行从0开始编号。

通常，所有数据都以字符串的形式从服务器传输。我们可以通过调用 PQgetvalue 函数得到数据的字符表示:

char *PQgetvalue(const PGresult *result, int tuple, int field);

如果需要提前知道 PQgetvalue 返回的字符串有多长，可以调用 PQgetlength：

int PQgetlength(const PGresult *result, int tuple, int field);

如前所述，元组(行)号和字段(列)号都从0开始。

处理NULL结果

在我们进一步讨论之前，有一个小问题必须解决。事实上，返回给我们的查询结果是编码在字符串中的，这意味着我们无法轻易区分空字符串和SQL NULL值。幸运的是，libpq库为我们提供了一个函数，我们可以调用该函数来确定结果集中某个字段的特定值是否为NULL:

int PQgetisnull(const PGresult *result, int tuple, int field);

当检索任何可能为NULL的字段时，我们应该调用 PQgetisnull。如果字段为NULL，返回1;0。上述示例程序的内循环就会变成下面这样：

for(n = 0; n < nfields; n++) {
    if(PQgetisnull(result, r, n))
        printf(" %s is NULL,", PQfname(result, n));
    else
         printf(" %s = %s(%d),", 
         PQfname(result, n), 
         PQgetvalue(result, r, n),
         PQgetlength(result, r, n));
}

打印查询结果

到目前为止，我们介绍的函数已经足够从PostgreSQL数据库中查询和提取数据了。如果我们想要做的只是打印结果，我们可以考虑利用libpq提供的打印函数，该函数以相当基本的形式输出结果集。这是 PQprint 函数，它将结果集格式化为类似于psql使用的表格形式，并将其发送到指定的输出流：

void PQprint(FILE *output, const PGresult *result, const PQprintOpt *options);

然而，PostgreSQL的维护者不再积极支持 PQprint，因此您不应该在生产代码中依赖它。它在开发和测试期间非常有用，也许在客户端程序中创建更复杂的显示结果的方式之前。

PQprint 参数是一个要打印的打开文件句柄(output)、一个结果集(result)和一个指向结构体的指针，该结构体包含控制打印格式的选项(options)。该结构如下：

struct {
    pqbool header; /* print out names of columns in a header */
    pqbool align; /* pad out the values to make them line up */
    pqbool html3; /* format as an HTML table */
    pqbool expanded; /* expand tables */
    pqbool pager; /* use pager for output if needed */
    char *fieldSep; /* field separator */
    char *tableOpt; /* options for HTML table - place in <TABLE …> */
    char *caption; /* HTML <caption> */
    char **fieldName; /* Replacement set of field names */
} PQprintOpt;

PQprintOpt 结构的成员相当简单。如果 header 成员设置为非零值，则导致输出表的第一行由字段名组成，可以通过设置字符串的fieldName 列表来覆盖这些字段名。

输出表中的每一行都由字段值组成，字段值由字符串 fieldSep 分隔，如果 align 为非零，则填充字段值以与其他行对齐。下面是PQprintOpt 的输出示例:

+-------------+-------+-----------+--------------+----------+----------+
| 3 | Miss | Alex | Nicetown | NT2 2TX | 010 4567 |

如果输出很长，可以将 pager 设置为非零值，要求对输出进行分页;也就是说，通过一个过滤器，在每一页左右暂停输出。如果expanded 设置为非零值，则输出格式更改为在一行中单独列出每行中的每个字段。

我们可以通过设置html3为非零值来生成适合网页包含的HTML输出。我们可以通过设置 tableOpt 和 caption 字符串来指定表选项和标题。下面是一个程序示例(print.c)，使用 PQprint 生成HTML输出:

#include <stdlib.h>
#include <libpq-fe.h>
int main()
{
    PGresult *result;
    PGconn *conn;
    conn = PQconnectdb("");
    if(PQstatus(conn) == CONNECTION_OK) {
        printf("connection made\n");
         result = PQexec(conn, "SELECT \* FROM customer WHERE town = 'Bingham'");
         { 
             PQprintOpt pqp;
             pqp.header = 1;
            pqp.align = 1;
            pqp.html3 = 1;
            pqp.expanded = 0;
            pqp.pager = 0;
            pqp.fieldSep = "";
            pqp.tableOpt = "align=center";
            pqp.caption = "Bingham Customer List";
            pqp.fieldName = NULL;
            printf("<html><head><title>Customers</title></head><body>\n");
            PQprint(stdout, result, &pqp);
            printf("</body></html>\n");
         }

    }
    PQfinish(conn);
    return EXIT_SUCCESS;

}

管理事务

有时，我们希望确保一组SQL命令作为一个组执行，以便在发生错误时一起对数据库进行更改，或者完全不进行更改。这种形式的查询分组被称为事务。

与标准SQL一样，我们可以使用libpq的事务支持来管理它。事务行为通过使用包含 BEGIN、COMMIT 和 ROLLBACK 的SQL语句调用PQexec 来实现:

PQexec(conn, "BEGIN WORK");
/* Make changes */
if(we changed our minds) {
     PQexec(conn, "ROLLBACK WORK");
}
else {
    PQexec(conn, "COMMIT WORK");
}

通过向 PQexec 传递适当的SQL查询字符串，我们的libpq程序可以使用所有功能。

使用游标

在编写实际应用程序时，我们可能需要处理大量数据。值得庆幸的是，PostgreSQL数据库能够存储包含大量行的表。然而，在处理产生大量数据的查询结果时，客户端应用程序及其操作环境就会对我们产生影响。桌面PC在处理一次SELECT返回的一百万个元组时很可能遇到麻烦。一个大的结果集可能会消耗大量的内存，如果应用程序是这样的话在网络上运行，可能会消耗大量的带宽，并需要大量的时间来传输。

我们真正需要做的是执行查询并一点一点地处理结果。例如，如果在我们的应用程序中，我们想要显示完整的客户列表，我们可以一次检索所有客户列表。但是，更聪明的做法是每次获取25个页面，并在我们的应用程序中逐页显示。

我们可以通过使用libpq的游标来实现这一点。游标是一种极好的通用方式，用于容纳返回未知数量的行。如果我们搜索特定的邮政编码，特别是用户提供的邮政编码，则不可能提前知道返回的是零、一还是多行。

一般来说，你应该避免编写假设SELECT语句返回单行或不返回任何行的代码，除非这条语句是一个简单的聚合操作，例如类型查询的SELECT count(*)，或者主键的SELECT，这些操作可以保证结果总是只返回一行。当有疑问时，使用游标。

为了演示如何处理从查询中返回的多行数据，我们将探索如何使用 FETCH 一次检索一个(或多个)行数据，并以与all-at-once SELECT语句相同的方式将列值接收到结果集中。

我们将逐步开发一个示例程序，该程序使用游标从bpfinal数据库逐页查询和处理客户列表。我们将声明一个游标，用于在返回的行集合中滚动。游标将充当我们的书签，我们将获取这些行，直到没有更多数据可用。要使用游标，必须声明它并指定与之相关的查询。我们只能在PostgreSQL事务中使用游标声明，所以我们也必须开始一个事务:

PQexec(conn, "BEGIN work");
PQexec(conn, "DECLARE mycursor CURSOR FOR SELECT ...");

现在可以开始检索结果行了。我们通过执行 FETCH 来实现这一点，每次提取尽可能多的数据行(包括剩余的所有行):

result = PQexec(conn, "FETCH 1 IN mycursor");
result = PQexec(conn, "FETCH 4 IN mycursor");
result = PQexec(conn, "FETCH ALL IN mycursor");

当检索到查询中的所有行时，结果集将表明它不包含任何行。使用完游标后，关闭游标并结束事务:

PQexec(conn, "COMMIT work");
PQexec(conn, "CLOSE mycursor");

我们花点时间研究一下使用游标时使用的一般结构:

#include <libpq-fe.h>
main()
{
     /* Connect to a PostgreSQL database */
     /* Create cursor for SQL SELECT statement */
     DO
     /* Fetch batch of query results */
     /* Process query results */
     UNTIL no more results
     /* close cursor */
     /* Disconnect from database */
}

分批获取结果

要修改程序以处理结果(比如一次处理4个结果)，我们需要能够判断何时取得了所有结果。当我们同时处理所有结果时，这很容易，因为PQntuples 会告诉我们集合中有多少个结果。如果我们一次获取4个结果，那么 PQntuples 将为每批结果返回4个结果，但最后一批结果将是小于4(可能为0)。这是我们将要使用的测试(在 cursor2.c 中)。由于我们的doSQL函数没有返回结果，我们将直接使用 PQexec 处理批次。

#include <stdlib.h>
#include <libpq-fe.h>
void printTuples(PGresult \*result)
{
    int r, n;
    int nrows = PQntuples(result);
    int nfields = PQnfields(result);
    printf("number of rows returned = %d\n", nrows);
    printf("number of fields returned = %d\n", nfields);
    for(r = 0; r < nrows; r++) {
        for(n = 0; n < nfields; n++)
            printf(" %s = %s(%d),", 
            PQfname(result, n), 
            PQgetvalue(result, r, n),
            PQgetlength(result, r, n));
            printf("\n");
        }
}
void doSQL(PGconn *conn, char *command)
{
    PGresult *result;
    printf("%s\n", command);
    result = PQexec(conn, command);
    printf("status is %s\n", PQresStatus(PQresultStatus(result)));
    printf("#rows affected %s\n", PQcmdTuples(result));
    printf("result message: %s\n", PQresultErrorMessage(result));
    switch(PQresultStatus(result)) {
     case PGRES_TUPLES_OK:
         printTuples(result);
         break;
     }
     PQclear(result);
}
    
int main()
{
    PGresult *result;
    PGconn *conn;
    int ntuples = 0;
    conn = PQconnectdb("");
     if(PQstatus(conn) == CONNECTION_OK) {
         printf("connection made\n");
     doSQL(conn, "BEGIN work");
     doSQL(conn, "DECLARE mycursor CURSOR FOR "
     "SELECT fname, lname FROM customer");
     do {
        result = PQexec(conn, "FETCH 4 IN mycursor");
        if(PQresultStatus(result) == PGRES_TUPLES_OK) {
        ntuples = PQntuples(result);
        printTuples(result);
        PQclear(result);
     }
     else ntuples = 0;
     } while(ntuples);
        doSQL(conn, "CLOSE mycursor");
        doSQL(conn, "COMMIT work");
    }
     else 
         printf("connection failed: %s\n", PQerrorMessage(conn));
         PQfinish(conn);
     return EXIT_SUCCESS;
}

处理二进制值本章中处理的所有数据都以字符串形式传输。我们创建了以字符串形式存储值的INSERT语句，并再次以字符串形式处理SELECT的结果。

您可能认为所有这些格式之间的转换都是浪费。使用二进制接口不是更好吗?如果想插入或选择一个浮点数，比如平均油耗，直接将它从C double类型的值转换到C double类型的值不是更好吗?

答案既是肯定的，也是否定的。PostgreSQL在libpq中确实有一些处理二进制值的能力，但好处可能相当少。目前，我们只能在使用游标并在其声明中包含二进制选项的情况下获取二进制值:

DECLARE mycursor BINARY CURSOR FOR...

然后 PQbinaryTuples 函数将确认从该游标获取返回的结果集包含二进制数据：

int PQbinaryTuples(const PGresult *result);

现在，在使用二进制元组的情况下，PQgetvalue 将返回一个指向字段二进制表示形式的指针，而不是字符串指针，这种二进制表示形式是服务器原生的二进制格式。

处理变长数据和多字节值的字节排序(如货币数据)等问题很快就会变得非常混乱。建议你在使用libpq时坚持使用字符串表示。使用嵌入式SQL可以更容易地处理二进制值。

异步工作

到目前为止，我们看到的所有例子都是在阻塞模式下使用libpq函数。这意味着我们的程序调用libpq函数并等待它返回值。正如我们之前看到的，我们可能会请求大量的数据，可能需要一段时间才能接收到。在单线程(普通)应用程序中，如果我们使用 PQexec，在此之前我们什么都不能做PQexec 回报。用户必须等待。如果不再需要查询结果，取消查询也很棘手。但对大多数程序来说，这就足够了，较短的等待不是问题。一次使用游标获取少量结果可能就足够了。在本节中，我们将介绍一种更高级的技术，如果需要，它可以让我们精确控制应用程序的行为。

阻塞的另一种方式，通常用于具有图形用户界面的应用程序，是在非阻塞模式下运行。在这种类型的程序中，当发生某些事情时，会向我们发送消息，我们的应用程序必须对这些消息作出响应。因此，在指向点击应用程序中，我们可能会被告知用户何时按下了鼠标按钮、何时在字段中输入数据、何时移动了窗口等等。

非阻塞程序的结构通常围绕着一个事件循环:

main()
{
    LOOP: {
        /* Wait for an event to occur */
        /* Find out what has event has occurred */
        switch(event type) {
            /* Process event */
        }
    }
}

在某些事件发生之前，我们基本上什么都不做。当我们收到某件事发生的通知时，我们准确地找出发生了什么，并执行适当的操作。

有时，我们使用的编程语言隐藏了事件循环，我们只需要在特定事件发生时调用我们编写的特定函数。这些函数通常被称为回调函数，构成应用程序核心的主循环在有话要说的时候会回调我们。例如，在Visual Basic中就是这样。

在libpq中，PostgreSQL对这种非阻塞编程方式有一定的支持。它被称为异步工作，因为在服务器上执行的数据库操作不与客户端应用程序同步，客户端应用程序可以执行其他任务，而不是简单的等待。相反，我们必须询问PostgreSQL是否发生了任何事情，以及什么时候发生了，我们恢复处理。

以异步模式执行查询

让我们看看如何在异步模式中执行查询。我们将使用 PQexec 用来完成工作的一对函数：PQsendQuery 和 PQgetResult。

我们的想法是调用 PQsendQuery 将我们的SQL发送到服务器。一旦它在路上，我们可以继续做其他事情。

int PQsendQuery(PGconn *conn, const char *query);

如果没有分派查询，PQsendQuery 返回0，并设置一个错误条件，我们可以通过 PQerrorMessage 检索。否则，返回1表示查询成功发送。如果服务器立即响应，结果集将一直等待我们，直到我们准备好检索它。准备好后，调用 PQgetResult 一次或多次，获取查询结果:

PGresult *PQgetResult(PGconn *conn);

PQgetResult 每次调用都会返回一个结果集，直到活动查询的所有结果都被返回为止。如果没有进一步的数据立即可用，结果集可能不包含元组，但这并不意味着所有数据都已经接收到。此时，PQgetResult 返回一个NULL指针。当我们调用 PQclear 时，PQgetResult 返回的每一个结果集都必须被清空。

为了让它工作，我们必须确保 PQsendQuery 和 PQgetResult 都不会阻塞，让我们的应用程序等待它们。为了帮助使用 PQsendQuery，我们可以通过调用 PQsetnonblocking 将连接本身设置为非阻塞状态:

int PQsetnonblocking(PGconn *conn, int arg);

为了防止 PQsendQuery 阻塞，我们使用一个非零参数调用 PQsetnonblocking。然后，如果 PQsendQuery 被阻塞，它将返回一个错误，应用程序可能在稍后重试。

如果成功，PQsetnonblocking 返回0，如果在改变连接模式时出现问题，则返回-1。我们可以通过调用 PQisnonblocking 来检查连接的阻塞模式:

int PQisnonblocking(const PGconn *conn);

如果连接处于非阻塞模式，我们会得到一个非零值。

运行这个非阻塞操作需要的编程是这样的(async1.c):

#include <stdlib.h>
#include <libpq-fe.h>
void printTuples(PGresult *result)
{
    int r, n;
    int nrows = PQntuples(result);
    int nfields = PQnfields(result);
    printf("number of rows returned = %d\n", nrows);
    printf("number of fields returned = %d\n", nfields);
    for(r = 0; r < nrows; r++) {
        for(n = 0; n < nfields; n++)
             printf(" %s = %s(%d),", 
             PQfname(result, n), 
             PQgetvalue(result, r, n),
             PQgetlength(result, r, n));
             printf("\n");
     }
}

int main()
{
    PGresult *result;
    PGconn *conn;
    conn = PQconnectdb("");
    if(PQstatus(conn) == CONNECTION_OK && PQsetnonblocking(conn,1) == 0) {
         printf("connection made\n");
         PQsendQuery(conn, "SELECT \* FROM customer");
         while(result = PQgetResult(conn)) {
             printTuples(result);
             PQclear(result);
         }
     }
    else 
        printf("connection failed\n");
    PQfinish(conn);
    return EXIT_SUCCESS;
}

该程序发送一个查询，然后无阻塞地收集结果。与前面使用游标的例子一样，我们现在没有简单的方法来确定总共返回多少行，但这在实践中很少成为问题。

仍然有一些罕见的情况下 PQgetResult 会阻塞，其中之一是在后端服务器繁忙时。有一些方法可以解决这个问题，使用一些较低级别的libpq函数。如果您需要对连接进行非常精确的控制，并且正在执行非常复杂的查询，这会导致服务器长时间繁忙，你可以使用表3中的函数。

请查看PostgreSQL文档，该文档包含在源代码分发版中，或在http://www.postgresql.org上了解有关使用 PQisBusy、PQconsumeInput 和 PQflush 函数的详细信息。

如果您的应用程序使用select系统调用(不要与SQL select语句混淆)来响应文件描述符或网络套接字上的读写事件，您也可以在select中包含PostgreSQL连接。为此，必须获取数据库连接正在使用的套接字。这可以从 PQsocket 中获得:

int PQsocket(const PGconn *conn);

函数 描述
int PQisBusy(PGconn *conn); 如果当前查询繁忙，则返回1，如果调用 PQgetResult，则会阻塞。
int PQflush(PGconn *conn); 尝试发送任何等待发送到服务器的未完成数据。如果成功清空了队列，或者队列已经为空，则返回0。
int PQconsumeInput(PGconn *conn); 将等待在数据库连接上读取的数据传输到内部的libpq数据结构中。它通常由 PQexec 之类的函数调用，但当需要控制应用程序的阻塞行为时，可以显式地调用它。

  • 必须没有未完成的数据要发送到服务器。我们可以通过调用 PQflush 直到它返回0来确保这一点。

  • 在调用 PQgetResult 之前，PQconsumeInput 必须从连接中读取数据。

取消异步查询

如果需要在读取所有结果之前取消查询，可以调用 PQrequestCancel，它会向服务器发送一个停止查询处理的通知:

int PQrequestCancel(PGconn *conn);

如果 PQrequestCancel 能够发送取消请求，则返回1;如果无法发送取消请求，则返回0，原因可能是查询已经完成。一个被取消的查询将显示为一个带有错误的结果集。无论 QrequestCancel 的结果如何，我们都可能来不及阻止结果出现，并且可能在结果集中看不到任何错误。

虽然 PQrequestCancel 在非阻塞模式下很有用，但也可以从信号处理程序中调用它来终止阻塞连接上的长时间运行的查询。

建立异步数据库连接

也可以使用 PQconnectStart 和 PQconnectPoll 以非阻塞的方式建立到数据库的初始连接:

PGconn *PQconnectStart(const char *conninfo);
PostgresPollingStatusType *PQconnectPoll(PGconn *conn);

PQconnectStart 类似于 PQconnectdb，不同的是它会在 conninfo 字符串中请求的连接建立之前立即返回。只要主机名参数不导致DNS查找，PQconnectStart 就不会阻塞。

在使用新连接之前，我们必须确保它已经准备好。首先，我们必须调用 PQstatus 来确保对 PQconnectStart 的调用不会失败，从而使连接的状态为 CONNECTION_BAD。然后，我们可以使用 PQconnectPoll 检查连接的条件，它不会阻塞。PQconnectPoll 返回的结果如下所示。

PGRES_POLLING_FAILED 连接失败

PGRES_POLLING_OK 连接已经建立

轮询的结果不是 PGRES_POLLING_FAILED，直到变成 PGRES_POLLING_OK，我们就可以以非阻塞的方式检测连接建立过程的结束。

下面是一个建立异步数据库连接的示例程序(async2.c):

#include <stdlib.h>
#include <libpq-fe.h>
int main()
{
    PGresult *result;
    PGconn *conn;
    /* Start an asynchronous connection */
    conn = PQconnectStart("");
    if(PQstatus(conn) == CONNECTION_BAD) {
        printf(" cannot start connect: %s\n", PQerrorMessage(conn));
    }
    else {
        /* do some work, calling PQconnectPoll from time to time */
        PostgresPollingStatusType status;
        do {
            printf("polling\n");
            status = PQconnectPoll(conn);
        }
        while(status != PGRES_POLLING_FAILED && status != PGRES_POLLING_OK);
        if(status == PGRES_POLLING_OK)
            printf("connection made!\n");
        else
            printf("connection failed: %s\n", PQerrorMessage(conn));
    }
    PQfinish(conn);
    return EXIT_SUCCESS;
}

当我们运行这个程序时，我们会看到许多轮询消息，在连接被报告为成功或失败之前：

$ PGDATABASE=bpfinal ./async2
polling
polling
...
connection made!

如果多次运行此程序，您可能会看到在建立连接之前出现不同数量的轮询消息，确切的数字取决于服务器响应连接请求所需的时间。

总结

在本章中，我们学习了如何用c语言创建PostgreSQL应用程序，了解了libpq库如何访问PostgreSQL的底层函数，允许我们通过网络连接本地机器或服务器上的数据库。我们已经使用了示例程序来建立和关闭连接，并执行SQL语句来查询、插入或更新数据库表中的行。

我们已经考虑了处理大量数据的问题，并研究了如何使用游标将查询结果编组为可管理的单元。我们已经了解了阻塞的问题，并考虑了创建应用程序的方法，以便在访问数据库服务器时继续为用户提供服务。
