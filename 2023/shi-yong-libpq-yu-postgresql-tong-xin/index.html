使用 LibPQ 与 PostgreSQL 通信

postgresql
postgresql
发布日期:   2023-01-12

libpq库是一个 PostgreSQL 客户端库，用于与 PostgreSQL 服务器通信。本章的目的是编写 C 程序以连接到 PostgreSQL 服务器并使用 C 编程语言执行查询。在本章中，我们将探索使用libpq 与 PostgreSQL 的通信，其中包括通过足够的 C 代码示例了解所有库函数及其用法。我们还将讨论libpq函数的阻塞和非阻塞行为。

[TOC]

1 连接和断开连接数据库

libpq库提供了多种功能和机制来连接到 PostgreSQL 服务器。我们将讨论与后端建立连接所涉及的每一个函数。应用程序可以与 PostgreSQL 数据库建立多个连接，但需要为每个连接维护PGconn指针。 PGconn指针是一个连接对象，也是一个用来建立连接的函数。它返回应用程序需要存储并用于后续函数查询数据库的指针。当不再需要与服务器的连接时必须关闭此对象，因为代码必须释放 PostgreSQL 运行时分配的资源。

使用 PQconnectdb

PQconnectdb函数是连接到 PostgreSQL 后端的最基本函数。这个函数只有一个参数： conninfo字符串。

PQconnectdb的语法如下：

PGconn *PQconnectdb (const char *conninfo);

  • conninfo字符串是一个以空格分隔的字符串，其中包含关键字值对，例如，keyword = ‘foo’。以下是所有conninfo关键字的列表：

  • hostaddr: 这是 PostgreSQL 服务器的 IP 地址

  • host: 这是 PostgreSQL 服务器的主机名，它可以是 Unix 域套接字的路径名称。默认值为/tmp目录，如果操作系统不支持 Unix 域套接字，则默认值为localhost。

  • port: 这是PostgreSQL服务器的端口号；默认端口为5432

  • dbname: 这是我们需要连接的数据库的名称；默认值是用户名

  • user: 这是连接的用户名；默认是操作系统用户

  • password: 这是认证期间使用的密码

  • connect_timeout: connect_timeout关键字显示PQconnectdb函数在放弃之前等待的时间

下面是一些不常用的选项：

  • client_encoding
  • options
  • application_name
  • fallback_application_name
  • keepalives
  • keepalives_idle
  • keepalives_interval
  • keepalives_count
  • tty
  • sslmode
  • disable
  • allow
  • verify-ca
  • verify-full
  • requiressl
  • sslcert
  • sslkey
  • sslrootcert
  • sslcrlrequirepeer
  • krbsrvname
  • gsslib
  • service

这些值的详细信息可以在http://www.postgresql.org/docs/9.4/static/libpqconnect.html#LIBPQ-PARAMKEYWORDS找到。

注意

如果您看到以下类型的错误，那么这是由于conninfo字符串中的错误造成的：

missing “=” after “port” in connection info string

使用 PQconnectdbParams

PQconnectdbParams函数是PQconnectdb函数的变体。与 PQconnectdb一样，此函数采用键和值的二维数组而不是单个键值对字符串。这个函数仅在 PQconnectdb 的函数参数上有所不同。

PQconnectdbParams的语法如下：

PGconn *PQconnectdbParams(const char **keywords,
                          const char **values,
                          int expand_dbname);

使用 PQsetdbLogin

PQsetdbLogin函数是连接到 PostgreSQL 服务器的另一种变体。此函数的参数数量有限，之前讨论的所有其他参数均用作默认值。 PQsetdbLogin的语法如下:

PGconn *PQsetdbLogin (const char *pghost,
                      const char *pgport,
                      const char *pgoptions,
                      const char *pgtty,
                      const char *dbName,
                      const char *login,
                      const char *pwd);

注意

每当您在连接到 PostgreSQL 时收到以下错误消息，这意味着 PostgreSQL 服务器没有运行，至少没有在5432端口上运行：

connection to database failed

could not connect to server: No such file or directory

Is the server running locally and accepting connections on Unix domain

socket “/tmp/.s.PGSQL.5432”?

您需要检查运行 PostgreSQL 服务器的主机和端口。

使用 PQsetdb

PQsetdb函数不是函数而是宏定义的，它使用默认的用户名和密码调用PQsetdbLogin函数。 PQsetdb的语法如下：

PGconn *PQsetdb(char *pghost,
                char *pgport,
                char *pgoptions,
                char *pgtty,
                char *dbName);

下面是使用前面讨论的所有方法建立连接的完整示例：

/*----------------------------------------------------------------
*
*connection.c
*Connecting PostgreSQL Server using libpq
*
*IDENTIFICATION
*connection.c
*
*----------------------------------------------------------------
*/
#include<stdio.h>
#include<sys/types.h>
#include<stdlib.h>
#include<limits.h>
#include "libpq-fe.h"
#include "libpq/libpq-fs.h"
charconninfo[] = "hostaddr = '127.0.0.1' port = '5432' dbname = 'testdb'";
char *keyword[] = {"hostaddr", "port", "dbname"};
char *value[] = {"127.0.0.1", "5432", "testdb"};
int main(int argc, char **argv)
{
    PGconn *conn; /* Connection Object */
    /* Connect database using PQsetdb */
    conn = PQsetdb(NULL, NULL, NULL, NULL, "testdb");
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed using PQsetdb");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    fprintf(stdout, "Connection to database established using PQsetdb\n");
    PQfinish(conn);
    /* Connect database using PQconnectdb */
    conn = PQconnectdb(conninfo);
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed using PQconnectdb\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    fprintf(stdout, "Connection to database established using
    PQconnectdb\n");
    PQfinish(conn);
    /* Connect database using PQconnectdbParams */
    conn = PQconnectdbParams((const char **)keyword, (const char **)value,1);
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed using
        PQconnectdbParams\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    fprintf(stdout, "Connection to database established using
    PQconnectdbParams\n");
    PQfinish(conn);
    /* Connect database using PQsetdbLogin */
    conn = PQsetdbLogin("127.0.0.1", "5432", NULL, NULL, "testdb", "foo","bar");
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed using PQsetdbLogin\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    fprintf(stdout, "Connection to database established using
    PQsetdbLogin\n");
    PQfinish(conn);
    return 0;
}

以下是编译所有示例的MakeFile的内容：

CC=gcc
POSTGRESQL_INCLUDE=/usr/local/pgsql.master/include
POSTGRESQL_LIB=/usr/local/pgsql.master/lib
CFLAGS=-I$(POSTGRESQL_INCLUDE)
LD_FLAGS=-L$(POSTGRESQL_LIB)
all:
$(CC) connection.c -o connection $(CFLAGS) $(LD_FLAGS) -lpq
$(CC) exec.c -o exec $(CFLAGS) $(LD_FLAGS) -lpq
$(CC) retrieve.c -o retrieve $(CFLAGS) $(LD_FLAGS) -lpq
connection:
$(CC) connection.c -o connection $(CFLAGS) $(LD_FLAGS) -lpq
exec:
$(CC) exec.c -o exec $(CFLAGS) $(LD_FLAGS) -lpq
retrieve:
$(CC) retrieve.c -o retrieve $(CFLAGS) $(LD_FLAGS) -lpq
清理:
rm -rf *.o connection exec retrieve

使用 PQfinish

PQfinish函数用于关闭与服务器的连接。它接受由PQconnectdb、 PQsetdbLogin或PQsetdb函数返回的连接对象PGconn指针。 PQfinish的语法如下：

Void PQfinish(PGconn *conn);

使用 PQreset

PQreset函数关闭之前的连接并建立新的连接。有时，我们在关闭前一个连接后需要一个新的连接。 PQreset函数不是调用PQfinish和

PQconnectdb，而是关闭连接并重新连接到服务器。

PQreset的语法如下：

Void PQreset(PGconn *conn);

2 异步建立连接

libpq库提供了异步连接到 PostgreSQL 数据库的函数；这些函数称为非阻塞函数。非阻塞函数不会阻塞并立即返回。 libpq库具有变体阻塞和非阻塞连接函数。我们已经讨论了阻塞函数，现在是时候讨论非阻塞函数了。

使用 PQconnectStartParams

PQconnectStartParams函数是PQconnectdbParams函数的非阻塞变体。完整的连接是在应用程序循环中而不是在库函数中进行的。 PQconnectStartParams的语法如下：

PGconn *PQconnectStartParams(const char **keywords,
                             const char **values,
                             int expand_dbname);

使用 PQconnectStart

PQconnectStart函数是PQconnectdb函数的非阻塞变体。与PQconnectStartParams函数一样，它也不会阻塞。

PQconnectdbParams的语法如下：

PGconn *PQconnectStart(const char *conninfo);

使用 PQconnectPoll

PQconnectPoll函数用于轮询一个连接的连接状态。 PQconnectPoll的语法如下：

PostgresPollingStatusType PQconnectPoll(PGconn *conn);

该函数可以返回以下值：

Functions Value
PGRES_POLLING_FAILED 0
PGRES_POLLING_READING 1
PGRES_POLLING_WRITING 2
PGRES_POLLING_OK 3
PGRES_POLLING_ACTIVE 4

使用 PQresetStart

PQresetStart函数是 PQreset 的非块变体。与其他非阻塞函数一样，它不会阻塞，整个连接是在一个应用程序循环中建立的，这意味着用户需要池化连接、重置或未连接的状态。

PQresetStart的语法如下：

Int PQresetStart(PGconn *conn);

使用 PQresetPoll

PQresetPoll函数用于轮询复位状态。 PQresetPoll的语法如下：

PostgresPollingStatusType PQresetPoll(PGconn *conn);

3 连接支持函数

除了核心库函数， libpq库还提供了一些实用函数。下面是一些实用函数。

使用 PQconninfoParse

PQconninfoParse函数解析字符串并返回PQconninfoOption指针。 PQconninfoParse的语法如下：

PQconninfoOption *PQconninfoParse(const char *conninfo,
                                        char **errmsg);

使用 PQpingParams

PQpingParams函数返回服务器状态。 PQpingParams的语法如下：

PGPing PQpingParams(const char **keywords,
                    const char **values,
                          int expand_dbname);

PQpingParams函数可以返回以下值：

  • PQPING_OK：表示服务器可用，即服务器正在接受连接
  • PQPING_REJECT：这意味着服务器正在拒绝连接
  • PQPING_NO_RESPONSE：表示服务器没有响应
  • PQPING_NO_ATTEMPT：这意味着有一个错误的参数并且没有进行连接尝试

4 执行查询

我们已经讨论了如何连接到 PostgreSQL 服务器；第二步是执行查询。 libpq库还提供了多个函数来执行查询，其方式与它提供多个函数以连接到 PostgreSQL 服务器的方式类似。

使用 PQexec

PQexec函数将命令（查询）发送到连接的 PostgreSQL 服务器。第一个参数是前面讨论的连接函数返回的同一个连接对象。 PQexec的语法如下：

PGresult *PQexec(PGconn *conn, const char *command);

使用 PQexecParams

在PQexec函数中，除了PGconn对象外，只有一个参数是命令字符串。所以，如果我们想发送一些值在命令字符串之外的命令，那么我们需要PQexecParams函数。 PQexecParams的语法如下：

PGresult *PQexecParams(PGconn *conn,
                        const char *command,
                        int nParams,
                        const Oid *paramTypes,
                        const char * const *paramValues,
                        const int *paramLengths,
                        const int *paramFormats,
                        insert resultFormat);

此函数采用额外参数，可用于在命令外部提供额外参数。它们如下：

  • nParams: 显示参数的总数
  • paramTypes: 显示参数的类型
  • paramValue: 显示参数的值数组
  • paramLengths: 显示参数数组的长度
  • paramFormats: 显示参数是二进制 (1) 还是文本 ( 0)

5 执行准备好的语句

PostgreSQL 支持准备好的语句。准备好的语句用于高效地重复执行相同或相似的查询。首先，查询以使用占位符的模板形式准备，并在执行期间替换值。在 PostgreSQL 中， $符号用于标识占位符。

使用 PQprepare

PQprepare函数提交请求以创建具有给定参数的准备语句并等待完成。 PQprepare的语法如下：

PGresult *PQprepare(PGconn *conn,
                    const char *stmtName,
                    const char *query,
                    int nParams,
                    const Oid *paramTypes);

PQprepare函数仅在协议版本 3.0 及更高版本中受支持。在版本低于 3.0 的情况下，该功能将失效。

PQprepare函数的参数如下：

  • stmtName：这是准备好的语句的名称
  • query：这是查询字符串
  • nParams：这是参数的个数
  • paramTypes：这显示了参数的数据类型

注意

如果stmtName为空字符串，将创建一个未命名的门户，并且将覆盖先前未命名的门户。

使用 PQexecPrepared

在使用 PQprepare 准备查询之后，我们可以使用 PQexecPrepared 执行查询。 PQexecPrepared函数执行先前准备好的语句。它的语法如下：

PGresult *PQexecPrepared(PGconn *conn,
                        const char *stmtName,
                        int nParams,
                        const char * const *paramValues,
                        const int *paramLengths,
                        const int *paramFormats,
                        int resultFormat);

PQexecPrepared函数的参数如下：

  • paramValues：显示参数数组的值
  • paramLengths：显示参数数组的长度
  • paramFormats：显示参数数组的格式
  • resultFormat ：以文本或二进制格式显示结果

PQexecPrepared函数执行使用PQprepare函数准备的查询。此函数使用stmtName来标识需要执行的查询。准备好的查询只解析和计划一次，但非准备的查询每次都解析和计划。

下面是一个完整的示例，说明如何使用讨论的连接和执行函数建立连接和执行查询：

/*----------------------------------------------------------------
*
*exec.c
*Executing queries using libpq
*
*IDENTIFICATION
*exec.c
*
*----------------------------------------------------------------
*/
#include <stdio.h>
#include <sys/types.h>
#include <stdlib.h>
#include <limits.h>
#include "libpq-fe.h"
#include "libpq/libpq-fs.h"
char conninfo[] = "hostaddr = '127.0.0.1' port = '5432' dbname = 'testdb'";
Oid paramTypes[] = {23, 25};
int paramLengths[] = {-1, 5};
int paramFormats[] = {0, 0};
int main(int argc, char **argv)
{
    PGconn *conn; /* Connection Object */
    PGresult *result;
    char *paramValues[] = {"2", "bar-2"};
    conn = PQconnectdb(conninfo);
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    /* Execute a query using PQexec */
    result = PQexec(conn, "CREATE TABLE foo (id INTEGER, name TEXT)");
    if (result == NULL || PQresultStatus(result) != PGRES_COMMAND_OK)
    {
        fprintf(stderr, "failed to execute query using PQexec\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    result = PQexec(conn, "INSERT INTO foo VALUES(1, 'bar-1')");
    if (result == NULL || PQresultStatus(result) != PGRES_COMMAND_OK)
    {
        fprintf(stderr, "failed to execute query using PQexec\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    /* Execute a query using PQexecParams */
    result = PQexecParams(conn, "INSERT INTO foo VALUES($1, $2)", 2, paramTypes, (const char * const*)paramValues, paramLengths, paramFormats,0);
    if (result == NULL || PQresultStatus(result) != PGRES_COMMAND_OK)
    {
        fprintf(stderr, "failed to execute query using PQexecParams\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    /* Execute a prepared query */
    result = PQprepare(conn, "test", "INSERT INTO foo VALUES($1, $2)", 2,
    paramTypes);
    paramValues[0] = "3";
    paramValues[1] = "bar-3";
    result = PQexecPrepared(conn, "test", 2, (const char *const*)paramValues, paramLengths, paramFormats,0);
    if (result == NULL || PQresultStatus(result) != PGRES_COMMAND_OK)
    {
        fprintf(stderr, "failed to execute query using PQexecPrepared\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    PQfinish(conn);
    return 0;
}

6 检索数据

现在，下一步是从 PostgreSQL 获取数据。以下是用于在查询执行后提取数据的函数。

使用 PQntuples

PQntuples函数返回结果中通常称为元组的行数。它的语法如下：

int PQntuples(const PGresult *res);

注意

大于 32 位的值可能会溢出。

使用 PQnfields

PQnfields 函数返回每行中的列数。 PQnfields的语法如下：

int PQnfields(const PGresult *res);

使用 PQfname

PQfname函数返回与column_number参数关联的列的名称。 PQfname的语法如下：

char *PQfname(const PGresult *res, int column_number);

使用 PQfnumber

PQfnumber函数返回与column_name参数关联的列号。 PQfnumber的语法如下：

int PQfnumber(const PGresult *res, const char *column_name);

以下是解释PQfnumber函数的代码片段：

ntuples = PQntuples(result);
for (i = 0; i < ntuples; i++)
{
    char *id, *name;
    /* Get the column number of column "id" and "name" */
    int id_col = PQfnumber(result, "id");
    int name_col = PQfnumber(result, "name");
    id = PQgetvalue(result, i, id_col);
    name = PQgetvalue(result, i, name_col);
    fprintf ("Data: id = %s, name = %s\n", id, name);
}

使用 PQftable

在引用多个表的复杂查询中，我们需要知道从中获取该列的表 OID。 PQftable函数获取结果集中该特定列的表的 OID。

PQftable的语法如下：

Oid PQftable(const PGresult *res, int column_number);

以下是解释PQftable函数的代码片段：

ntuples = PQntuples(result);
for (i = 0; i < ntuples; i++)
{
    char *id, *name;
    int id_col = PQfnumber(result, "id");
    int name_col = PQfnumber(result, "name");
    /* Get Table oid of column "id" */
    int oid = PQftable(result, id_col);
    fprintf(stdout, "Table oid is = %d \n", oid);
}

使用 PQftablecol

PQftablecol函数返回表中的列号。 PQftablecol的语法如下：

int PQftablecol(const PGresult *res, int column_number);

使用 PQfformat

PQfformat函数返回列的格式。 PQfformat的语法如下：

int PQfformat(const PGresult *res, int column_number);

使用 PQftype

PQftype函数返回列的类型。 PQftype的语法如下：

Oid PQftype(const PGresult *res, int column_number);

使用 PQfmod

PQfmod函数返回列的类型修饰符。 PQfmod的语法如下：

int PQfmod(const PGresult *res, int column_number);

使用 PQfsize

PQfsize函数返回列的大小。 PQfsize的语法如下：

int PQfsize(const PGresult *res, int column_number);

使用 PQbinaryTuples

PQbinaryTuples函数返回结果是二进制还是文本（1 表示二进制，0 表示文本） 。 PQbinaryTuples的语法如下：

int PQbinaryTuples(const PGresult *res);

使用 PQgetvalue

PQgetvalue函数返回单行列的值。 PQgetvalue的语法如下：

char *PQgetvalue(const PGresult *res, int row_number, int column_number);

使用 PQgetisnull

PQgetisnull函数用于检查特定行的列的值是否为空。如果值为空，则无需从表中获取值。 PQgetisnull的语法如下：

int PQgetisnull(const PGresult *res, int row_number, int column_number);

以下是解释PQgetisnull函数的代码片段：

ntuples = PQntuples(result);
for (i = 0; i < ntuples; i++)
{
    char *id, *name;
    int id_col = PQfnumber(result, "id");
    int name_col = PQfnumber(result, "name");
    int oid = PQftable(result, id_col);
    fprintf(stdout, "id is %s \n", PQgetisnull(result, i, id_col)? "NULL":
    "NOT NULL");
    fprintf(stdout, "name is %s \n", PQgetisnull(result, i, name_col)?
    "NULL": "NOT NULL");
}

使用 PQgetlength

PQgetlength函数返回特定行的列的实际长度。 PQgetlength的语法如下：

int PQgetlength(const PGresult *res, int row_number, int column_number);

使用 PQnparams

PQnparams函数返回准备语句的参数总数。PQnparams的语法如下：

int PQnparams(const PGresult *res);

使用 PQparamtype

PQparamtype函数返回指定语句参数的数据类型。 PQparamtype的语法如下：

Oid PQparamtype(const PGresult *res, int param_number);

下面是从 PostgreSQL 服务器中选择数据的例子。以下程序显示了PQexec、 PQntuples、 PQfnumber和PQfinish函数的用法：

/*----------------------------------------------------------------
*
*retrieve.c
*Retrieving data using libpq
*
*IDENTIFICATION
*retrieve.c
*
*----------------------------------------------------------------
*/
#include <stdio.h>
#include <sys/types.h>
#include <stdlib.h>
#include <limits.h>
#include "libpq-fe.h"
#include "libpq/libpq-fs.h"
char conninfo[] = "hostaddr = '127.0.0.1' port = '5432' dbname =
'postgres'";
int main(int argc, char **argv)
{
    PGconn *conn; /* Connection Object */
    PGresult *result;
    int ntuples;
    int i;
    conn = PQconnectdb(conninfo);
    if (PQstatus(conn) == CONNECTION_BAD)
    {
        fprintf(stderr, "connection to database failed\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    /* Execute a query using PQexec */
    result = PQexec(conn, "SELECT id, name FROM foo LIMIT 3");
    if (result == NULL || PQresultStatus(result) != PGRES_TUPLES_OK)
    {
        fprintf(stderr, "failed to execute query using PQexec\n");
        fprintf(stderr, "%s", PQerrorMessage(conn));
        return -1;
    }
    ntuples = PQntuples(result);
    fprintf(stdout, "%d tuples returns\n", ntuples);
    for (i = 0; i < ntuples; i++)
    {
        char *id, *name;
        int id_col = PQfnumber(result, "id");
        int name_col = PQfnumber(result, "name");
        if (id_col < 0 || name_col < 0)
        {
            fprintf(stderr, "invalid result\n");
            break;
        }
        id = PQgetvalue(result, i, id_col);
        name = PQgetvalue(result, i, name_col);
        fprintf("Data: id = %s, name = %s\n", id, name);
    }
    PQfinish(conn);
    return 0;
}

7 使用状态函数

有一些错误处理函数或状态函数可用于检查函数的结果。

使用 PQresultStatus

PQresultStatus函数返回一个命令的状态。这个函数对于在执行查询后获取结果状态非常重要。这个函数是程序逻辑的基础，因为程序应该根据结果状态采取行动。 PQtresultStatus的语法如下：

ExecStatusType PQresultStatus(const PGresult *res);

以下是解释PQresultStatus函数的代码片段：

/* Execute a query using PQexec */
result = PQexec(conn, "SELECT id, name FROM foo LIMIT 3");
switch(PQresultStatus(result))
{
    case PGRES_COMMAND_OK:/* 查询成功，无数据返回 */
    break;
    case PGRES_TUPLES_OK:/* 查询正常且数据可用 */
    break;
    case PGRES_EMPTY_QUERY:/* 查询字符串为空 */
    break;
    case PGRES_COPY_OUT:/* 开始从服务器端传输数据 */
    break;
    case PGRES_COPY_IN:/*&nbsp;&nbsp;开始向服务器传输数据 */
    break;
    case PGRES_BAD_RESPONSE:/* 来自服务器的无效响应 */
    break;
    case PGRES_NONFATAL_ERROR: /* 出现通知或警告 */
    break;
    case PGRES_FATAL_ERROR:/* 发生Fata错误. */
    break;
    case PGRES_COPY_BOTH:/* 开始向/从服务器传输数据 */
    break;
    default:/* 未知错误 */
    break;
}

使用 PQresStatus

此函数将PQresultStatus函数返回的状态代码转换为字符串。 PQresStatus的语法如下：

char *PQresStatus(ExecStatusType status);

以下代码片段执行查询并获取返回结果的状态PQexec使用PQresStatus函数：

/* Execute a query using PQexec */
result = PQexec(conn, "SELECT id, name FROM foo LIMIT 3");
fprintf(stderr, "failed to execute query: %s\n",
PQresStatus(PQresultStatus(result)));

failed to execute query: PGRES_FATAL_ERROR

默认情况下， libpq是线程安全的，可以使用PQisthreadsafe检查 libpq 的线程安全状态。应该使用特殊的编译器命令行选项来编译基于线程的应用程序。
