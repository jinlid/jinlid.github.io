搭建Prometheus+Grafana的云平台监控系统

postgresql Linux
监控
发布日期:   2023-01-30

Prometheus是一套开源的监控&报警&时间序列数据库的组合，起始是由SoundCloud公司开发的。现在最常见的 Docker、Mesos、Kubernetes 容器管理系统中，通常会搭配Prometheus 进行监控。

Prometheus 基本原理是通过 HTTP 协议周期性抓取被监控组件的状态，这样做的好处是任意组件只要提供 HTTP 接口就可以接入监控系统，不需要任何 SDK 或者其他的集成过程。这样做非常适合虚拟化环境比如 VM 或者Docker。

输出被监控组件信息的 HTTP 接口被叫做 exporter。目前互联网公司常用的组件大部分都有exporter 可以直接使用，比如 Varnish、Haproxy、Nginx、MySQL、Linux 系统信息 (包括磁盘、内存、CPU、网络等等)，具体支持的源看：https://github.com/prometheus。

[TOC]

1安装配置 Prometheus 监控服务

1.1 添加用户和组

groupadd -f prometheus
useradd -g prometheus -d /home/prometheus -m -s /bin/bash prometheus
cd /home/prometheus

1.2 下载prometheus-2.41.0

wget https://github.com/prometheus/prometheus/releases/download/v2.41.0/prometheus-2.41.0.linux-amd64.tar.gz
tar -xf prometheus-2.41.0.linux-amd64.tar.gz
chown prometheus:prometheus prometheus-2.41.0.linux-amd64
cd prometheus-2.41.0.linux-amd64

1.3 配置prometheus.yml

此处使用默认配置

# my global config
global:
  scrape_interval: 15s # Set the scrape interval to every 15 seconds. Default is every 1 minute.
  evaluation_interval: 15s # Evaluate rules every 15 seconds. The default is every 1 minute.
  # scrape_timeout is set to the global default (10s).

# Alertmanager configuration
alerting:
  alertmanagers:
    - static_configs:
        - targets:
          # - alertmanager:9093

# Load rules once and periodically evaluate them according to the global 'evaluation_interval'.
rule_files:
  # - "first_rules.yml"
  # - "second_rules.yml"

# A scrape configuration containing exactly one endpoint to scrape:
# Here it's Prometheus itself.
scrape_configs:
  # The job name is added as a label `job=<job_name>` to any timeseries scraped from this config.
  - job_name: "prometheus"

    # metrics_path defaults to '/metrics'
    # scheme defaults to 'http'.

    static_configs:
      - targets: ["localhost:9090"]

默认端口号是9090，可以修改。

1.4 配置自启动服务

#新增文件内容 /usr/lib/systemd/system/prometheus.service
[Unit]
Description=Prometheus Monitoring System
Documentation=Prometheus Monitoring System

[Service]
ExecStart=/home/prometheus/prometheus-2.41.0.linux-amd64/prometheus --config.file=/home/prometheus/prometheus-2.41.0.linux-amd64/prometheus.yml --web.listen-address=:9090

[Install]
WantedBy=multi-user.target

添加到开机自启动

systemctl daemon-reload
systemctl start prometheus
systemctl enable prometheus

访问http://192.168.137.129:9090/targets，192.168.137.129是我虚拟机IP，State显示UP表示部署正常。

2 配置node_exporter监控服务器状态

2.1 下载node_exporter

cd /home/prometheus
wget https://github.com/prometheus/node_exporter/releases/download/v1.5.0/node_exporter-1.5.0.linux-amd64.tar.gz
tar -xf node_exporter-1.5.0.linux-amd64.tar.gz

2.2 启动node_exporter

cd node_exporter-1.5.0.linux-amd64
./node_exporter &

2.3 在prometheus.yml添加agent节点

  - job_name: "node129"
    static_configs:
      - targets: ["192.168.137.129:9100"]

注意缩进。配置完毕后，执行kill -s SIGHUP，对配置文件进行重新加载。

kill -s SIGHUP `ps -ef|grep prometheus |grep -v grep|awk '{print $2}'`

再次访问http://192.168.137.129:9090/targets，显示多了node129，State是Up表示正常。

3 配置postgres_exporter监控postgresql服务状态

3.1 下载postgres_exporter

cd /home/prometheus
wget https://github.com/prometheus-community/postgres_exporter/releases/download/v0.11.1/postgres_exporter-0.11.1.linux-amd64.tar.gz
tar -xf postgres_exporter-0.11.1.linux-amd64.tar.gz
cd postgres_exporter-0.11.1.linux-amd64

3.2 启动postgres_exporter

启动之前设置一个DATA_SOURCE_NAME环境变量，它指定数据库连接字符串。

export DATA_SOURCE_NAME="postgresql://postgres:123456@192.168.137.129:5432/postgres?sslmode=disable"
./postgres_exporter &

启动成功，postgres_exporter监听9187端口。

3.3 在prometheus.yml添加agent节点

  - job_name: "pg96"
    static_configs:
      - targets: ["192.168.137.129:9187"]

注意缩进。配置完毕后，执行kill -s SIGHUP，对配置文件进行重新加载。

kill -s SIGHUP `ps -ef|grep prometheus |grep -v grep|awk '{print $2}'`

再次访问http://192.168.137.129:9090/targets，显示多了postgres，State是Up表示正常。

4 配置grafana

4.1 下载grafana-enterprise-9.3.6

wget https://dl.grafana.com/enterprise/release/grafana-enterprise-9.3.6.linux-amd64.tar.gz
tar -xf grafana-enterprise-9.3.6.linux-amd64.tar.gz

4.2 启动grafana

cd grafana-9.3.6
bin/grafana-server &

访问http://192.168.137.129:3000，默认端口3000，可以在conf/defaults.ini修改，默认用户和密码是admin/admin。

登录之后，首先需要配置数据源，确定显示的数据是从哪个数据源获取的。

点击Add data source，

在次选择Prometheus数据源。填写url，指定数据源。

点击Save & test，显示Data source is working则正常。

然后是Create Dashboard，点击Import。

在Import via grafana.com输入模板ID 12633（linux主机详情），模板ID 9628（postgresql数据库监控），点击load。

也可以从模板导入，模板文件及模板ID可以在Grafana Lab网站上下载或查找：https://grafana.com/grafana/dashboards/

在导入的选项里。可以设置面板的名字、文件夹。DS_PROMETHEUS这里选择Prometheus。

导入完成不久您将看到漂亮的监控。

至此Prometheus+Grafana监控平台已经搭好。
