iptables介绍及常用命令

docker
操作系统
发布日期:   2021-08-13

防火墙是网络安全的基本工具。通过在服务器和外部访客之间建立过滤机制，防火墙在网络层面上实现了安全防范。Linux的防火墙工具是IP Tables，这套防火墙系统甚至被作为很多其他专业网络设备的核心。

1 介绍

iptables主机型防火墙工作在2，3，4层，对TCP/IP数据包进行过滤和限制。属于过滤型防火墙。（除非编译内核才可以使iptables支持7层）。

1.2 iptables的缺点

  1. 防火墙虽然可以过滤互联网的数据包，但却无法过滤内部网络的数据包。因此若有人从内部网络攻击时，防火墙没有作用。
  2. 电脑本身的操作系统亦可能因一些系统漏洞，使入侵者可以利用这些漏洞饶过防火墙过滤。从而入侵电脑。
  3. 防火墙无法有效阻挡病毒攻击，尤其隐藏在数据中的病毒。
  4. 正常情况下，所有互联网的数据包软件都应经过防火墙的过滤，这将造成网络交通的瓶颈。例如在攻击性数据包出现时，攻击者不时寄出数据包，让防火墙疲于过滤数据包，而使一些合法数据包软件亦无法正常进出防火墙。

PS：没有绝对安全的操作系统，虽然防火墙有这些缺点，但还是能阻挡大多数来自外网的攻击。

1.3 核心原理

1.3.1 四个表

（1）filter

其中默认包含了3个链，分别是FORWARD、INPUT和OUTPUT。

  1. FORWARD：途经本机的数据包进行过滤
  2. INPUT：访问本机的数据包进行过滤
  3. OUTPUT：本机访问互联网的数据包进行过滤

总结：根据规则来处理数据包，如转或者丢。（实现主机型防火墙的主要表；内核模块iptables_filter）。

（2）nat

  1. PREROUTING：修改目的地址
  2. POSTROUTING：修改源地址
  3. OUTPUT：主要是控制内部

总结：转换地址的表（改IP，改端口；当网关使用的Linux保护内部流量；内核模块是iptables_nat）。

（3）mangle

  1. PREROUTING
  2. POSTROUTING
  3. INPUT
  4. OUTPUT
  5. FORWARD

总结：流量整形的表；路由标记用的表；改TOS，TTL，Mark标记策略路由等；内核模块iptables_mangle。

（4）raw

  1. PREROUTING
  2. OUTPUT

总结：新表；数据包跟踪；内核模块：iptables_raw。

表的应用顺序：

raw>>mangle>>nat>>filter。

1.3.2 iptables的五个链

  • PREROUTING预路由
  • POSTROUTING已路由
  • INPUT入站
  • OUTPUT出站
  • FORWARD转发

FORWARD链中定义的规则作用于那些需要转发到另一个网络接口的数据包。INPUT链中定义的规则作用于发送到本机的数据包。相对应的，OUTPUT链中定义的规则作用域从本机发送出去的数据包。通常定义filter表就可以迎合大部分的安全需求，因为这个表包含了包过滤器的所有内容。

三种报文流向

流入本机：POSTROUTING –> INPUT –> 用户空间进程

流出本机：用户空间进程 –> OUTPUT –> POSTROUTING

转发：POSTROUTING –> FORWARD –> POSTROUTING

链之间的顺序：

入站>>转发>>出站

规则的应用顺序：

ACL逐条匹配，匹配即停止。（除了LOG）

2 防火墙设置

2.1 iptables语法：

iptables -t 表名 管理选项 [][][][][链名] [匹配条件] [-j 控制类型]

  1. 表名

    filter（不写默认）、nat、mangle、raw。

  2. 管理选项

    操作方式如插入，删除，查看等。

  3. 链名

    一般是FORWORD、INPUT和OUTPUT中的一个。

  4. 匹配条件

    数据包特征ip，端口等。

  5. 控制类型

    用于定义策略，数据包处理方式，filter表中共有9个不同的策略可供使用，但最常用的只有4个：

    • ACCEPT：表示允许包通过。
    • DROP：丢弃一个包。
    • REJECT：会在丢失的同时返回一条ICMP错误消息。
    • LOG：扮演了计事员的角色记录包的信息，并把它们写入日志。

itables命令最常用的5个选项分别是-F、-P、-I、-A、-D和-L。在大部分情况下，管理员只需要这5个选项就可以完成防火墙的规则配置。
iptables的常用选项：

选项含义

  • -F：清除链中所有的规则（-X是清理自定义的链，用的少；-Z清零规则序号）
  • -P：为链设置一条默认策略（或者说目标）
  • -I：在链的开头（或指定序号）插入一条规则
  • -A：为链增加一条规则说明
  • -D：删除链内指定序号（或内容）的一条规则
  • -L：查看当前表中的链和规则
  • -n：以数字形式显示地址、端口等信息
  • -v：以更详细的方式显示规则信息
  • –line-numbers：查看规则时，显示规则的序号（方便之处，通过需要来删除和插入）

查看当前的防火墙设置

iptables -nvL

以上target目标，prot端口，opt选项，source源地址，destination目标地址。

简单的例子：

  1. 从192.168.138.81过来的包DROP掉

尝试从192.168.138.81ping，发现连接卡住

  1. 从192.168.138.82过来的包REJECT掉

尝试从192.168.138.82ping，发现连接报错：Destination Port Unreachable

使用tcpdump抓包发现，192.168.138.81有去无回，从192.168.138.82有去有回

2.4 删除规则

iptables提供了-D选项来删除链规则，有2种不同的语法用于删除一条规则。

iptables -D chain rule-specification 
iptables -D chain rulenum

第1种语法使用规则描述来匹配某条链路规则。为此。用户必须一字不差地照搬当初使用-A选项添加使用的描述。如删除对lo环回接口的规则设置。

iptables -D INPUT -i lo -p ALL -j ACCEPT

第2种语法删除编号为2的链规则

iptables -D INPUT 2

清空默认表（也就是filter表）中的数据

iptables -F

清空默认表中INPUT链的规则

iptables -F INPUT

2.5 基本的匹配条件

基本匹配条件：无需加载模块，由iptables/netfilter自行提供

  • -s, –source address[/mask] [,…]：源ip地址或者不连续的ip地址

  • -d, –destination address[/mask] [,…]：目标ip地址或者不连续的ip地址

  • -p, –protocol protocol ：指定协议，可使用数字如0（all）

    protocol：tcp,udp,icmp,icmpv6,udplite,ah,sctp,mh or “all”，参考：/etc/protocols

  • -i, –in-interface name：报文流入的接口：只能应用于数据报文流入环节，只应用于INPUT、FORWARD、PREROUTING链

  • -o, –out-interface name：报文流出的接口：只能应用于数据报文流出环节，只应用于FORWARD、OUTPUT、PREROUTING链

不让别人ping自己

iptables -t filter -I INPUT -p icmp -j REJECT

删除该规则

iptables -t filter -D INPUT -p icmp -j REJECT

下面这条命令将INPUT链的默认策略更改为DROP（丢弃），通常对服务器而言，将所有的链的默认策略设置为DROP事一个好的建议。

iptables -P INPUT DROP

防止本地也无法连接，首先将本地地址添加进来。

执行完这条命令后，所有试图同本机建立连接的努力都会失败，因为所有从“外部”到达防火墙包都被丢弃了，甚至连使用环回接口ping自己都不行

ping localhost

假设当前防火墙所在的主机是一台Web服务器，为此应该允许外部主机能够连接到80端口（对应HTTP服务器）和22端口（对应SSH服务）。使用iptables -A命令添加链规则，基本语法如下：

iptables -A chain-name -i interface -j target

下面这条命令添加了一条INPUT链的规则，允许所有的lo接口的连接请求，这样防火墙就不会阻止“自己连接自己”的行为了。

iptables -A INPUT -i lo -p ALL -j ACCEPT

很多时候，管理员想要做的并不仅仅是把别人挡在门外，还希望直到有哪些人正在试图访问服务器。下面这条命令给INPUT链添加了一条LOG（日志记录）策略。

iptables -A INPUT -i eht0 -j LOG

默认情况下，防火墙记录到的访问信息被保存在/var/log/messages中。这是一个文本文件，可以使用任何文本查看命令查看。事实上，/var/log/messages记录了系统中的大部分行为，这是Linux主要的系统日志文件。
其中比较常用的记录字段有IN（接收数据包的网络接口）、SRC（数据包来源的IP地址）、DST（数据包的目的IP地址）及开头的日期和时间等。

建议使用黑名单模式，允许所有的网段都能连接，最后剩下的都拒绝。

2.6 扩展匹配条件

扩展匹配条件：需要加载(/usr/lib64/xtables/*.so)，方可生效

2.6.1 隐式扩展

iptables在使用-p选项指明了特定的协议时，无需再用-m选项指明扩展模块的扩展机制，不需要手动加载扩展模块

tcp协议的扩展选项

  • –source-port,–sport port[:port]：匹配报文源端口，可为端口连续范围

  • –destination-port,–dport port[:port]：匹配报文源端口，可为端口连续范围

  • –tcp-flags mask comp

    mask 需检查的标志位列表，用,分隔，例如SYN,ACK,FIN,RST

    comp 在mask列表中必须为1的标志位列表，无指定则必须为0，分隔tcp协议的扩展选项

    –tcp-flags SYN,ACK,FIN,RST SYN 表示要检查的标志位为YN,ACK,FIN,RST四个，其中SYN必须为1，余下的必须为0，第一次握手

    –tcp-flags SYN,ACK,FIN,RST SYN,ACK 第二次握手

    错误包 –tcp-flags ALL ALL

    –tcp-flags ALL NONE

    –syn：用于匹配第一次握手，相当于–tcp-flags SYN,ACK,FIN,RST SYN

udp协议的扩展选项

  • –source-port,–sport port[:port]：匹配报文的源端口或端口范围
  • –destination-port,–dport port[:port]：匹配报文的目标端口或端口范围

icmp协议的扩展选项

  • –icmp-type {type[/code]|typename}

    type/code

    0/0 echo-repl icmp应答

    8/0 echo-request icmp请求

一些例子：

网站出现问题时，管理员可以简单地使用ping命令确定这台服务器是否还在运行。如果服务器拥有两个网路接口eth0和ppp0，分别对应内部网络和Internet。那么一些网络管理员会倾向于将ppp0设置为丢弃外部地ping请求。

iptables -A INPUT -i eth0 -p icmp --icmp-type 8 -j ACCEPT

命令地-p选项指定该规则匹配协议ICMP，紧跟地–icmp-type指定了ICMP的类型代码。ping命令对用的类型代码是8.
接下来的两条命令增加了22端口和80端口的访问许可。注意这次-p选项指定的协议类型是tcp，这时因为SSH服务和HTTP服务都是基于TCP协议的。

iptables -A INPUT -i eth0 -p tcp --dport 22 -j ACCEPT 
iptables -A INPUT -i eht0 -p tcp --dport 80 -j ACCEPT

如果网络接口eth0通向Internet，那么将SSH服务项全世界开放有时不那么让人放心，有些管理员可能希望更进一步，将SSH服务设置为只对本地网络的用户开放。下面的设置指定只有10.68.24.0/24这个网络中的主机可以访问22端口。

iptables -A INPUT -i eth0 -s 10.68.24.0/24 -p tcp --dport 22 -j ACCEPT
