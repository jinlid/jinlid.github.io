Docker介绍及常用命令

docker
操作系统
发布日期:   2021-08-13

Docker简介 Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中,然后发布到任何流行的Linux或Windows操作系统的机器上,也可以实现虚拟化,容器是完全使用沙箱机制,相互之间不会有任何接口。

[TOC]

1 安装docker

卸载旧版本

yum remove docker \
           docker-client \
           docker-client-latest \
           docker-common \
           docker-latest \
           docker-latest-logrotate \
           docker-logrotate \
           docker-engine

使用docker阿里云仓库安装

yum-config-manager \
--add-repo \
http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

安装 Docker Engine-Community

yum install docker-ce docker-ce-cli containerd.io

启动docker

systemctl start docker

通过运行 hello-world 映像来验证是否正确安装了 Docker Engine-Community

docker run hello-world

2 Docker镜像

Docker运行容器前需要本地存在对应的镜像，如果镜像不在本地，Docker会从镜像仓库下载（默认是Docker Hub公共注册服务器中的仓库）

检索可用镜像

docker search centos

拉取操作系统镜像

docker pull centos

列出本地镜像

docker images

REPOSITORY TAG IMAGE ID CREATED SIZE
opengauss 1.1.0 371bb14eed42 4 hours ago 651MB
centos 7.6.1810 f1cb7c7d58b7 2 years ago 202MB
…
列出的字段信息

  • REPOSITORY – 来自哪个仓库
  • TAG – 用来标记来自同一个仓库的不同镜像
  • IMAGE ID – 镜像的ID（惟一）
  • CREATED – 创建时间
  • SIZE – 镜像大小

创建并启动容器：

docker run -it --name centos1 centos /bin/bash

如果不指定具体的标记，则默认使用latest标记信息。

创建镜像

创建镜像有很多方法，用户可以从Docker Hub获取已有镜像并更新，也可以利用本地文件系统创建一个。
1 修改已有镜像，将容器制作成镜像

docker commit -m '镜像描述' -a '制作者' 容器名 镜像名
  • -m 来指定提交的说明信息，跟版本控制工具一样
  • -a 可以指定更新的用户信息
  • 容器名 指定用来创建镜像的容器的ID
  • 镜像名 指定目标镜像的仓库名和tag信息

创建成功后会返回这个镜像的ID信息。之后，可以使用新的镜像来启动容器

docker run -it --name 新容器 新镜像 /bin/bash

2 利用Dockerfile来创建镜像
新建一个目录和一个DockerFile

[root@host docker]# mkdir pg9.6
[root@host docker]# cd pg9.6/
[root@host pg9.6]# touch Dockerfile
DockerFile中每一条指令都创建镜像的一层，例如：
[root@host2 pg9.6]# cat Dockerfile

This is a comment

FROM pg96
MAINTAINER Docker jinli 845401160@qq.com
RUN su - postgres
DockerFile基本语法

  • 使用#来注释
  • FROM指令告诉Docker使用哪个镜像作为基础
  • 接着是维护者信息
  • RUN开头的指令会在创建中运行

编写完成Dockerfile后使用docker build来生成镜像

[root@host pg9.6]# docker build -t=”pg9.6:v2” .
Sending build context to Docker daemon 2.048kB
Step 1/3 : FROM pg96
—> 85c794950329
Step 2/3 : MAINTAINER Docker jinli 845401160@qq.com
—> Running in 39f9100e0615
Removing intermediate container 39f9100e0615
—> 2242bebfb210
Step 3/3 : RUN su - postgres
—> Running in c5229326c565
Last login: Mon May 31 06:01:21 UTC 2021 on pts/2

  • -t标记来添加tag，指定新的镜像的用户信息。
  • “.”是Dockerfile所在的路径（当前目录），也可以替换为一个具体的Dockerfile路径。
    可以看到build进程在执行操作。它要做的第一件事情就是上床这个Dockerfile内容，因为所有的操作都要依据Dockerfile来进行。然后，Dockerfile中的指令被一条条的执行。每一步都创建了一个容器，在容器中执行指令并提交修改。当所有的指令都执行完毕之后，返回了最终的镜像id。所有的中间步骤所产生的容器都被删除和清理了。
    此外还可以利用ADD命令复制本地文件到镜像；用EXPOSE命令来向外开放端口；用CMD命令来描述容器启动后运行的程序等。
    docker images查看新建的镜像REPOSITORY和TAG都为

[root@host pg9.6]# docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
2242bebfb210 9 minutes ago 258MB
ops_info latest 34ea62b0988c 5 weeks ago 2.65GB
pg96 latest 85c794950329 3 months ago 258MB
postgresql10.0 latest f7f1ca43042f 4 months ago 234MB
docker tag命令来修改镜像的标签
[root@host pg9.6]# docker tag 2242bebfb210 pg9.6:v2
[root@host pg9.6]# docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
pg9.6 v2 2242bebfb210 10 minutes ago 258MB
ops_info latest 34ea62b0988c 5 weeks ago 2.65GB
pg96 latest 85c794950329 3 months ago 258MB
postgresql10.0 latest f7f1ca43042f 4 months ago 234MB

上传镜像

docker push命令把自己创建的镜像上传到仓库中来共享。

docker push 镜像名

打包镜像

将制作好的镜像打成tar包文件到本地

docker save -o tar包的名字 镜像名

导入镜像

从到处的本地文件中再导入到本地镜像库

docker load < tar 包所在路径

这将导入镜像及其相关的元数据信息（包括标签等）

移除镜像

docker rmi 镜像名

在删除镜像之前需要先用docker rm删除依赖这个镜像的所有容器

3 Docker容器

简单的说，容器是独立于运行的一个或一组应用，以及它们的运行环境。对应的，虚拟机可以理解为模拟运行的一整套操作系统（提供了运行态环境和其他系统环境）和跑在上面的应用。

启动容器

启动容器分为两种，一种是基于镜像新建一个容器并启动，另外一个是将在停止状态的容器重新启动。
1 新建并启动

docker run -t -i 容器名 /bin/bash
  • -t 选项让Docker分配一个伪终端并绑定到容器的标准输入上
  • -i 则让容器的标准输入保持打开
    当使用ducker run来创建容器时，Docker在后台运行的标准操作包括：
  • 检查本地是否存在指定的镜像，不存在就从共有仓库下载
  • 利用镜像创建并启动一个容器
  • 分配一个文件系统，并在只读的镜像层外面挂载一层可读写层
  • 从宿主主机配置的网桥接口中桥接一个虚拟接口到容器中去
  • 从地址池配置一个ip地址给容器
  • 执行用户指定的应用程序
  • 执行完毕后容器被终止

2 启动已停止的容器

docker start 容器名

查看容器状态

docker ps -a
  • -a 参数可以查看包含已经停掉的容器

停止容器

使用docker stop命令来停止一个运行的容器，另外，当第一次创建并启动一个终端的容器，用户可以通过exit或者Crtl+D来退出终端时，所创建的容器立刻终止。

docker stop 容器名

重启容器

docker restart 容器名

进入容器

在使用-d参数时，容器启动后会进入后台。某些时候需要进入容器进行操作，有很多办法，包括使用docker attac命令或nsenter工具等。
attach命令

docker run -idt 镜像名
docker attach 容器名

导出容器

docker export 容器名 > tar包名

这样讲导出容器快照到本地文件。

导入容器快照

docker import tar包名 容器名

跟docker import 镜像名区别在于容器快照文件将丢弃所有的历史记录和元数据信息（即仅保存容器当时的快照状态），而镜像存储文件将保存完成记录，体积也要大。此外，从容器导入快照文件导入时可以成功指定标签等元数据信息。

删除容器

docker rm 容器名

如果删除一个正在运行的容器，可以添加-f参数。Docker会发送SIGKILL信号给容器。
