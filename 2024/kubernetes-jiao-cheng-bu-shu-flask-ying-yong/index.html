Kubernetes教程--部署Flask应用

容器
Kubernetes
发布日期:   2024-06-07

[TOC]

1. 环境准备

确保已安装以下工具：

  • kubectl（Kubernetes 命令行工具）
  • Docker（本地构建镜像）
  • 可用的 Kubernetes 集群（如 Minikube、Kind 或云服务商提供的集群）

2. 创建命名空间

创建 postgres-ns 命名空间用于部署 PostgreSQL 数据库：

# postgres-ns.yaml
apiVersion: v1
kind: Namespace
metadata:
  name: postgres-ns
  labels:
    name: postgres-ns

执行命令：

kubectl apply -f postgres-ns.yaml

kubectl get namespaces | grep postgres
# 输出应包含：postgres-ns       Active

3. 创建数据库凭证

创建 Secret 存储 PostgreSQL 密码、用户名和数据库名：

kubectl create secret generic postgres-secret -n postgres-ns \
  --from-literal=POSTGRES_PASSWORD=123456 \
  --from-literal=POSTGRES_USER=postgres \
  --from-literal=POSTGRES_DB=mydb

验证：

kubectl get secret -n postgres-ns
# 应看到 postgres-secret 已创建

4. 部署 PostgreSQL 数据库

创建 StatefulSet 部署 PostgreSQL：

# postgres-statefulset.yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: postgres
  namespace: postgres-ns
spec:
  serviceName: postgres
  replicas: 2
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:13
        env:
        - name: POSTGRES_PASSWORD
          valueFrom:
            secretKeyRef:
              name: postgres-secret
              key: POSTGRES_PASSWORD
        - name: POSTGRES_USER
          valueFrom:
            secretKeyRef:
              name: postgres-secret
              key: POSTGRES_USER
        - name: POSTGRES_DB
          valueFrom:
            secretKeyRef:
              name: postgres-secret
              key: POSTGRES_DB
        ports:
        - containerPort: 5432
          name: postgres

执行命令：

kubectl apply -f postgres-statefulset.yaml -n postgres-ns

5. 暴露 PostgreSQL 服务

创建 NodePort 类型的 Service 对外暴露数据库：

# postgres-service.yaml
apiVersion: v1
kind: Service
metadata:
  name: postgres
  namespace: postgres-ns
  labels:
    app: postgres
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    name: postgres
    targetPort: 5432
  type: NodePort

执行命令：

kubectl apply -f postgres-service.yaml -n postgres-ns

验证：

kubectl get pods -n postgres-ns --show-labels
kubectl get svc -n postgres-ns

# 获取 NodePort
NODE_PORT=$(kubectl get svc postgres -n postgres-ns -o jsonpath='{.spec.ports[0].nodePort}')
echo "PostgreSQL NodePort: $NODE_PORT"

# 测试连接（从集群外部）
psql -h <节点IP> -p $NODE_PORT -U postgres

6. 构建 Flask 应用镜像

创建 Dockerfile 构建 Flask 应用镜像：

# 使用官方 Python 运行时作为基础镜像
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 先复制依赖文件，利用 Docker 缓存加速构建
COPY requirements.txt .

# 安装依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制整个应用目录
COPY . .

# 暴露 Flask 默认端口
EXPOSE 5000
 
# 关闭 Flask 调试模式（通过环境变量控制）
ENV FLASK_DEBUG=0

# 使用 Gunicorn 启动（替代 app.run()）
CMD ["gunicorn", "--bind", "0.0.0.0:5000", "flask_movie.app:app"]

构建镜像：

docker build -t flask-movie-app .

7. 部署 Flask 应用到 Kubernetes

创建 Deployment 部署 Flask 应用：

apiVersion: apps/v1
kind: Deployment
metadata:
  name: flask-movie
  namespace: postgres-ns
spec:
  replicas: 1
  selector:
    matchLabels:
      app: flask-movie
  template:
    metadata:
      labels:
        app: flask-movie
    spec:
      containers:
      - name: flask-movie
        image: flask-movie-app
        imagePullPolicy: Never
        ports:
        - containerPort: 5000

执行命令：

kubectl apply -f flask-movie-deployment.yaml
kubectl get pods -n postgres-ns -o wide  # 查看应用 Pod 状态

8. 为 Flask 应用创建 NodePort 服务

创建 Service 资源将 Flask 应用暴露到集群外部：

apiVersion: v1
kind: Service
metadata:
  name: flask-movie-service
  namespace: postgres-ns
spec:
  selector:
    app: flask-movie  # 匹配 Pod 的 label
  ports:
    - protocol: TCP
      port: 5000      # Service 端口
      targetPort: 5000 # Pod 端口
  type: NodePort      # 或 LoadBalancer/ClusterIP

执行命令：

kubectl apply -f flask-movie-service.yaml

验证服务：

kubectl get svc -n postgres-ns
# 输出示例：
NAME                  TYPE       CLUSTER-IP      EXTERNAL-IP   PORT(S)          AGE
flask-movie-service   NodePort   10.110.185.40   <none>        5000:30000/TCP   11m
postgres              NodePort   10.99.247.133   <none>        5432:32039/TCP   23h

9. 访问 Flask 应用

通过任意节点 IP 和 NodePort 访问应用：

# 获取集群节点 IP
NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[?(@.type=="ExternalIP")].address}')

# 若没有 ExternalIP，使用 InternalIP
if [ -z "$NODE_IP" ]; then
  NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
fi

echo "访问地址: http://$NODE_IP:30000"

10. 故障排除

若部署失败，可使用以下命令排查问题：

# 查看 StatefulSet 状态
kubectl get statefulset -n postgres-ns
kubectl describe statefulset postgres -n postgres-ns

# 查看事件日志（按时间排序）
kubectl get events -n postgres-ns --sort-by='.metadata.creationTimestamp'

# 查看 Pod 详细信息和日志
kubectl describe pod <pod-name>
kubectl logs <pod-name>

11. 注意事项

  1. 镜像拉取策略：

    • imagePullPolicy: Never 仅适用于本地已存在的镜像。
    • 若使用远程镜像仓库，需改为 IfNotPresent 或 Always，并配置镜像仓库认证。

  2. 数据库连接：

    • Flask 应用需配置正确的数据库连接字符串，格式为：

      postgresql://<用户名>:<密码>@<服务名>.<命名空间>.svc.cluster.local:<端口>/<数据库名>
# 示例：
SQLALCHEMY_DATABASE_URI = 'postgresql://postgres:123456@postgres.postgres-ns.svc.cluster.local:5432/mydb'

  3. 生产环境优化：

    • 添加持久化存储（PV/PVC）给 PostgreSQL。
    • 使用 ConfigMap 管理应用配置。
    • 添加 Health Check（存活和就绪探针）。
    • 使用 Ingress 替代 NodePort 暴露服务。

总结

通过本教程，你学会了：

  1. 在 Kubernetes 中创建命名空间和 Secret。
  2. 使用 StatefulSet 部署有状态应用（PostgreSQL）。
  3. 构建 Flask 应用 Docker 镜像。
  4. 部署 Flask 应用到 Kubernetes。
  5. 基本的故障排查方法。

后续可扩展内容：添加 Service 暴露 Flask 应用、配置自动扩缩容、集成 CI/CD 流水线。
