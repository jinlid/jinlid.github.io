Kubernetes教程--集群部署

容器
Kubernetes
发布日期:   2024-06-07

[TOC]

1. 环境准备（所有节点执行）

集群部署前需统一节点基础配置，避免因环境差异导致部署失败。以下操作需在 master、node01、node02 三个节点 分别执行。

1.1 系统基础配置

1.1.1 设置主机名

为每个节点配置唯一主机名，便于后续识别和管理：

# master 节点执行
echo 'master' > /etc/hostname

# node01 节点执行
echo 'node01' > /etc/hostname

# node02 节点执行
echo 'node02' > /etc/hostname

1.1.2 配置 hosts 文件

手动绑定节点 IP 与主机名，避免依赖外部 DNS，确保节点间通信稳定：

cat >> /etc/hosts <<EOF
192.168.137.226 master
192.168.137.227 node01
192.168.137.228 node02
EOF

1.1.3 关闭防火墙

Kubernetes 组件间通信需开放多个端口，简化配置直接关闭防火墙（生产环境可按需配置规则）：

# 停止防火墙服务
systemctl stop firewalld
# 禁止防火墙开机自启
systemctl disable firewalld

1.2 禁用 Swap 分区

Kubernetes 要求禁用 Swap，避免内存交换影响集群性能和稳定性：

# 临时禁用 Swap（立即生效，重启后失效）
swapoff -a

# 永久禁用 Swap（注释 /etc/fstab 中的 Swap 配置）
sed -i '/\<swap\>/ s/^/#/' /etc/fstab

2. 安装容器运行时（所有节点执行）

Kubernetes 1.24+ 推荐使用 containerd 作为容器运行时，需先安装 Docker（依赖其基础环境），再配置 containerd。

2.1 安装 Docker 并配置镜像加速

# 安装 Docker（默认安装最新稳定版，如需指定版本可加版本号，如 docker-ce-24.0.6）
yum install docker-ce -y

# 配置 Docker 镜像加速（解决公网镜像拉取慢问题）
tee > /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": [
    "https://docker.registry.cyou",
    "https://mirror.aliyuncs.com",
    "https://dockerproxy.com",
    "https://mirror.baidubce.com",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}
EOF

# 启动 Docker 并设置开机自启
systemctl enable docker --now

2.2 安装并配置 containerd

# 安装指定版本 containerd（1.6.6 版本兼容性较好）
yum install containerd.io-1.6.6 -y

# 生成 containerd 默认配置文件
mkdir -p /etc/containerd
containerd config default > /etc/containerd/config.toml

# 修改配置：启用 SystemdCgroup（适配 Kubernetes 服务管理）
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml

# 配置 crictl（Kubernetes 官方工具，用于操作 containerd）
cat > /etc/crictl.yaml << EOF
runtime-endpoint: unix:///run/containerd/containerd.sock
image-endpoint: unix:///run/containerd/containerd.sock
timeout: 10
debug: false
EOF

# 重启 containerd 并设置开机自启
systemctl enable containerd --now

3. 配置 Kubernetes 依赖内核参数（所有节点执行）

加载网络相关内核模块，启用 IPv4 转发等关键参数，确保 Kubernetes 网络正常工作。

3.1 加载 br_netfilter 模块

# 临时加载模块（立即生效）
modprobe br_netfilter

# 永久加载模块（重启后仍生效）
echo "br_netfilter" > /etc/modules-load.d/k8s.conf

3.2 配置内核参数

# 写入内核参数配置文件
cat > /etc/sysctl.d/k8s.conf << EOF
net.bridge.bridge-nf-call-ip6tables=1  # 让 iptables 处理 IPv6 流量
net.bridge.bridge-nf-call-iptables=1   # 让 iptables 处理 IPv4 流量
net.ipv4.ip_forward=1                  # 启用 IPv4 转发（容器网络依赖）
EOF

# 应用配置（立即生效）
sysctl -p /etc/sysctl.d/k8s.conf

4. 安装 Kubernetes 组件（所有节点执行）

通过 YUM 源安装 kubeadm（集群初始化工具）、kubelet（节点管理组件）、kubectl（集群命令行工具）。

4.1 配置 Kubernetes YUM 源（阿里云源，国内访问快）

cat > /etc/yum.repos.d/kubernetes.repo <<EOF
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64/
enabled=1
gpgcheck=1
gpgkey=https://mirrors.aliyun.com/kubernetes/yum/doc/yum-key.gpg
       https://mirrors.aliyun.com/kubernetes/yum/doc/rpm-package-key.gpg
repo_gpgcheck=1
EOF

# 清理 YUM 缓存并生成新缓存
yum clean all
yum makecache

4.2 安装指定版本的 Kubernetes 组件

# 安装 1.27.0 版本
yum install -y kubelet-1.27.0 kubeadm-1.27.0 kubectl-1.27.0

# 设置 kubelet 开机自启（暂不启动，初始化后自动启动）
systemctl enable kubelet

5. 初始化 Master 节点（仅 master 节点执行）

Master 节点是集群控制平面核心，需通过 kubeadm 初始化，生成集群配置和节点加入令牌。

5.1 准备初始化配置文件

# 生成默认初始化配置模板
kubeadm config print init-defaults > kubeadm.yaml

# 编辑配置文件（关键参数已标注，按实际环境修改）
vi kubeadm.yaml

最终配置文件内容如下（重点关注 advertiseAddress、podSubnet、serviceSubnet）：

apiVersion: kubeadm.k8s.io/v1beta3
bootstrapTokens:
- groups:
  - system:bootstrappers:kubeadm:default-node-token
  token: abcdef.0123456789abcdef  # 默认令牌，可自定义
  ttl: 24h0m0s
  usages:
  - signing
  - authentication
kind: InitConfiguration
localAPIEndpoint:
  advertiseAddress: 192.168.137.226  # Master 节点 IP，需修改为实际 IP
  bindPort: 6443  # API Server 默认端口
nodeRegistration:
  criSocket: unix:///var/run/containerd/containerd.sock  # containerd 连接地址
  imagePullPolicy: Never  # 不自动拉取镜像（后续手动拉取国内源镜像）
  name: master  # Master 节点主机名，需与实际一致
  taints: null  # 初始不设置污点，如需禁止调度可后续添加
---
apiServer:
  timeoutForControlPlane: 4m0s
apiVersion: kubeadm.k8s.io/v1beta3
certificatesDir: /etc/kubernetes/pki  # 证书存储目录
clusterName: kubernetes  # 集群名称
controllerManager: {}
dns: {}
etcd:
  local:
    dataDir: /var/lib/etcd  # etcd 数据存储目录
imageRepository: registry.k8s.io  # 镜像仓库（后续手动替换为国内源）
kind: ClusterConfiguration
kubernetesVersion: 1.27.0  # Kubernetes 版本，需与安装版本一致
networking:
  dnsDomain: cluster.local  # DNS 域名
  podSubnet: 10.244.0.0/16  # Pod 网段（需与 Calico 配置一致）
  serviceSubnet: 10.96.0.0/12  # Service 网段
scheduler: {}

5.2 手动拉取 Kubernetes 镜像（国内源）

默认 registry.k8s.io 镜像仓库国内访问慢，通过华为云源拉取镜像并重新打标签：

# 1. 拉取控制平面组件镜像（仅 Master 节点需执行）
# kube-apiserver
ctr images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-apiserver:v1.27.0
ctr images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-apiserver:v1.27.0 registry.k8s.io/kube-apiserver:v1.27.0

# kube-controller-manager
ctr images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-controller-manager:v1.27.0
ctr images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-controller-manager:v1.27.0 registry.k8s.io/kube-controller-manager:v1.27.0

# kube-scheduler
ctr images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-scheduler:v1.27.0
ctr images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-scheduler:v1.27.0 registry.k8s.io/kube-scheduler:v1.27.0

# etcd
ctr images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/etcd:3.5.7-0
ctr images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/etcd:3.5.7-0 registry.k8s.io/etcd:3.5.7-0

# 2. 拉取所有节点都需要的镜像（Master + Worker 节点均需执行）
# kube-proxy（需在 k8s.io 命名空间下）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-proxy:v1.27.0
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/kube-proxy:v1.27.0 registry.k8s.io/kube-proxy:v1.27.0

# pause（基础沙箱镜像，需在 k8s.io 命名空间下）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/pause:3.6
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/pause:3.6 registry.k8s.io/pause:3.6

# coredns（DNS 组件，需在 k8s.io 命名空间下）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/coredns/coredns:v1.10.1
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/registry.k8s.io/coredns/coredns:v1.10.1 registry.k8s.io/coredns/coredns:v1.10.1

5.3 执行 Master 节点初始化

# 基于配置文件初始化集群
kubeadm init --config kubeadm.yaml

# 配置 kubectl 命令行工具（让当前用户拥有集群操作权限）
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# 验证 kubectl 配置（输出 Master 节点信息即成功）
kubectl get node

6. 安装网络插件 Calico（仅 master 节点执行）

Kubernetes 需网络插件实现 Pod 间通信，Calico 是常用的高性能网络插件，需确保其网段与 kubeadm.yaml 中的 podSubnet 一致。

6.1 下载并修改 Calico 配置文件

# 创建目录并下载配置文件
mkdir -p ~/k8s/calico && cd ~/k8s/calico
curl -O https://raw.githubusercontent.com/projectcalico/calico/v3.26.3/manifests/calico.yaml

# 修改 Pod 网段（与 kubeadm.yaml 中的 podSubnet 保持一致，默认是 192.168.0.0，需改为 10.244.0.0）
sed -i -e "s/192.168.0.0/10.244.0.0/g" calico.yaml

6.2 拉取 Calico 镜像（所有节点执行）

# calico-cni（网络接口插件）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/calico-cni:v3.26.3-rancher1
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/calico-cni:v3.26.3-rancher1 docker.io/calico/cni:v3.26.3

# calico-node（节点网络代理）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/mirrored-calico-node:v3.26.3
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/mirrored-calico-node:v3.26.3 docker.io/calico/node:v3.26.3

# calico-kube-controllers（Calico 控制器）
ctr -n k8s.io images pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/mirrored-calico-kube-controllers:v3.26.3
ctr -n k8s.io images tag swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/rancher/mirrored-calico-kube-controllers:v3.26.3 docker.io/calico/kube-controllers:v3.26.3

6.3 部署 Calico 网络插件

# 应用 Calico 配置
kubectl apply -f calico.yaml

# 验证 Calico 部署状态（所有 Pod 状态为 Running 即成功）
kubectl get pods -n calico-system

7. 加入 Worker 节点

Worker 节点需通过 Master 生成的 “加入命令” 加入集群，实现统一管理。

7.1 获取节点加入命令（仅 master 节点执行）

# 生成加入命令（包含令牌和 CA 证书哈希，有效期 24 小时）
kubeadm token create --print-join-command

执行后会输出类似如下命令（需复制到 Worker 节点执行）：

kubeadm join 192.168.137.226:6443 --token abcdef.0123456789abcdef --discovery-token-ca-cert-hash sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

7.2 执行加入命令（仅 node01、node02 执行）

# 先重置节点（避免之前操作残留配置影响）
kubeadm reset

# 执行从 Master 节点复制的加入命令（替换为实际输出的命令）
kubeadm join 192.168.137.226:6443 --token abcdef.
