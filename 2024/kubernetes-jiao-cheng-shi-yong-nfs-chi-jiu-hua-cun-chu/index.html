Kubernetes教程--使用NFS持久化存储

容器
Kubernetes
发布日期:   2024-06-17

本教程将详细介绍如何在 Kubernetes 中使用 NFS 作为持久化存储，包括 PV (PersistentVolume)、PVC (PersistentVolumeClaim) 的创建和使用，以及 Deployment 挂载 PVC 的完整流程。

[TOC]

1. NFS 服务器配置

NFS（Network File System）是一种分布式文件系统协议，允许客户端通过网络访问服务器上的文件。

步骤：

  1. 安装 NFS 服务

    yum install nfs-utils -y
systemctl enable nfs-server

  2. 创建共享目录并配置权限

    mkdir /home/jinli/nfs/data -p
chown -R jinli: /home/jinli/nfs

  3. 配置 NFS 导出规则

    cat > /etc/exports <<EOF
/home/jinli/nfs/data 192.168.137.226/24(rw,no_root_squash)
EOF
    • rw：允许读写访问
    • no_root_squash：不将 root 用户映射为匿名用户

  4. 启动 NFS 服务

    systemctl start nfs-server

  5. 验证共享目录

    su - jinli
touch /home/jinli/nfs/data/a  # 创建测试文件

2. NFS 客户端挂载测试

在 Kubernetes 节点上验证 NFS 挂载：

# 在节点上执行
showmount -e 192.168.137.226  # 查看NFS服务器共享列表

mkdir /home/jinli/nfs/data -p
chown -R jinli:jinli /home/jinli/nfs

mount -t nfs 192.168.137.226:/home/jinli/nfs/data /home/jinli/nfs/data

df -h  # 查看挂载结果
ll /home/jinli/nfs/data  # 应看到测试文件a

3. 创建 Kubernetes PersistentVolume (PV)

PV 是集群中的存储资源，由管理员预先配置。

# pv-nfs.yml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: pv001
  labels:
    pvname: pv001
spec:
  capacity:
    storage: 2Gi
  accessModes:
    - ReadWriteOnce  # 单节点读写
  persistentVolumeReclaimPolicy: Recycle  # 删除PVC后自动清理数据
  nfs:
    path: /home/jinli/nfs/data
    server: 192.168.137.226

应用并验证：

kubectl apply -f pv-nfs.yml
kubectl get pv  # 状态应为Available

4. 创建 PersistentVolumeClaim (PVC)

PVC 是用户对存储的请求，用于绑定 PV。

# pvc-nfs.yml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: myclaim01
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 2Gi
  selector:
    matchExpressions:
      - key: pvname
        operator: In
        values:
          - pv001  # 匹配PV的标签

应用并验证：

kubectl apply -f pvc-nfs.yml
kubectl get pvc  # 状态应为Bound
kubectl get pv  # PV状态应为Bound

5. 在 Pod 中使用 PVC

通过 Deployment 创建 Pod，并挂载 PVC。

# pv-deployment.yml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: pv-deploy
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nfs-pod
  template:
    metadata:
      labels:
        app: nfs-pod
    spec:
      containers:
      - name: myfrontend
        image: nginx:latest
        volumeMounts:
        - mountPath: "/var/www/html"  # 容器内挂载路径
          name: mypd
      volumes:
      - name: mypd
        persistentVolumeClaim:
          claimName: myclaim01  # 引用PVC

应用并验证：

kubectl apply -f pv-deployment.yml
kubectl get pods  # 确保Pod状态为Running

# 验证数据共享
cp /etc/profile /home/jinli/nfs/data/  # 在NFS服务器添加文件
kubectl exec <pod-name> -- ls /var/www/html  # 应看到a和profile文件

6. 数据持久化验证

删除 Pod 后重建，数据仍然存在：

kubectl delete pod <pod-name>  # 删除Pod
kubectl get pods  # 等待新Pod启动

kubectl exec <new-pod-name> -- ls /var/www/html  # 数据仍然存在

7. 清理资源

按顺序删除资源：

kubectl delete -f pv-deployment.yml  # 删除Deployment
kubectl delete -f pvc-nfs.yml  # 删除PVC

# 检查PV状态变为Released后，自动清理数据（Recycle策略）
kubectl get pv  # 最终状态变为Available
ls /home/jinli/nfs/data  # NFS目录应被清空

关键点说明

  1. PV 与 PVC 绑定：通过标签选择器（selector）匹配 PV 和 PVC。
  2. 访问模式：
    • ReadWriteOnce：单节点读写
    • ReadOnlyMany：多节点只读
    • ReadWriteMany：多节点读写
  3. 回收策略：
    • Recycle：删除 PVC 后自动清理数据（NFS 支持）
    • Retain：保留数据，需手动清理
    • Delete：删除 PV 时自动删除存储（云存储常用）
  4. NFS 优势：支持多节点挂载，适合需要共享存储的场景。

8. 常见问题排查

  1. PVC 状态为 Pending：
    • PV 容量不足
    • 标签选择器不匹配
    • NFS 服务器不可达
  2. Pod 无法启动：
    • 检查 NFS 服务器 IP 和路径是否正确
    • 确认 NFS 服务正常运行
    • 检查防火墙是否开放 NFS 端口（默认 2049）

通过以上步骤，你已成功实现 NFS 与 Kubernetes 的集成，为 Pod 提供了持久化存储。
