Harbor私有镜像仓库使用教程

Harbor
Kubernetes
发布日期:   2024-06-07

[TOC]

1. 环境准备

Harbor 依赖 Docker 和 Docker Compose 运行，需先完成基础工具安装，确保后续部署顺利。

1.1 安装 Docker Compose

Docker Compose 用于编排 Harbor 依赖的多个容器（如 Nginx、PostgreSQL、Redis 等），执行以下命令完成安装：

# 下载 Docker Compose 二进制文件（v2.37.1 版本，可根据需求替换为最新版）
wget https://github.com/docker/compose/releases/download/v2.37.1/docker-compose-linux-x86_64 -O /usr/local/bin/docker-compose

# 为二进制文件添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证安装结果，若输出版本信息则说明安装成功
docker-compose version

2. 安装 Harbor

Harbor 提供离线安装包，包含所有依赖组件，无需在线拉取，适合内网环境部署。本节将完成 Harbor 下载、证书配置（HTTPS 必需）、配置文件修改及安装操作。

2.1 下载并解压 Harbor 安装包

# 下载 Harbor v2.6.1 离线安装包（可从 GitHub 官网获取最新版链接）
wget https://github.com/goharbor/harbor/releases/download/v2.6.1/harbor-offline-installer-v2.6.1.tgz

# 解压安装包到指定目录（此处以 /opt 为例，可自定义路径）
tar xf harbor-offline-installer-v2.6.1.tgz -C /opt/

# 进入 Harbor 安装目录，后续操作均在此目录执行
cd /opt/harbor/

2.2 配置 HTTPS 证书

Harbor 支持 HTTP 和 HTTPS 访问，若用于生产环境或 Kubernetes 集群，建议启用 HTTPS（Kubernetes 默认要求镜像仓库使用安全连接）。以下步骤将生成自签 CA 根证书和服务器证书（包含 IP SANs，避免证书验证失败）。

2.2.1 创建证书存储目录

先创建统一的证书目录，便于管理：

mkdir -p /data/ssl/
cd /data/ssl/

2.2.2 生成 CA 根证书

CA 根证书用于签署服务器证书，客户端需信任此 CA 才能通过证书验证：

# 1. 生成 CA 私钥（无密码，避免 Harbor 启动时需手动输入密码）
openssl genrsa -out ca.key 4096

# 2. 生成 CA 根证书（有效期 10 年，subj 字段可根据实际情况修改）
openssl req -x509 -new -nodes -key ca.key -sha256 -days 3650 -out ca.pem \
  -subj "/C=CN/ST=shanxi/L=xian/O=test/OU=test/CN=Harbor CA"
# 说明：subj 字段含义：C（国家）、ST（省份）、L（城市）、O（公司）、OU（部门）、CN（CA 名称）

2.2.3 生成服务器证书（包含 IP SANs）

服务器证书用于 Harbor 的 Nginx 服务，需包含 Harbor 服务器的 IP（或域名）作为「主题备用名称（SANs）」，否则客户端通过 IP 访问时会提示证书验证失败。

# 1. 生成服务器私钥
openssl genrsa -out server.key 4096

# 2. 创建 openssl 配置文件，指定 SANs 信息（关键：解决 x509: cannot validate certificate 错误）
cat > openssl.cnf << EOF
[req]
default_bits = 4096
distinguished_name = req_distinguished_name
req_extensions = req_ext
prompt = no  # 关闭交互式输入，直接使用配置文件中的默认值

[req_distinguished_name]
C = CN                    # 国家（默认值）
ST = shanxi               # 省份（可自定义）
L = xian                  # 城市（可自定义）
O = test                  # 公司（可自定义）
OU = test                 # 部门（可自定义）
CN = 192.168.137.220      # 通用名称（必须为 Harbor 服务器的 IP 或域名）

[req_ext]
subjectAltName = @alt_names  # 启用 SANs 扩展

[alt_names]
IP.1 = 192.168.137.220    # Harbor 服务器的 IP（必须包含，可添加多个 IP/域名）
EOF

# 3. 生成证书请求文件（CSR）
openssl req -new -key server.key -out server.csr -config openssl.cnf

# 4. 用 CA 根证书签署服务器证书（有效期 10 年）
openssl x509 -req -in server.csr -CA ca.pem -CAkey ca.key -CAcreateserial \
  -out server.crt -days 3650 -sha256 -extensions req_ext -extfile openssl.cnf

2.2.4 验证证书有效性

执行以下命令，确认服务器证书包含 IP SANs：

# 查看服务器证书详情，搜索「Subject Alternative Name」字段
openssl x509 -in /data/ssl/server.crt -noout -text | grep -A 5 "Subject Alternative Name"
# 正常输出应包含：X509v3 Subject Alternative Name: IP Address:192.168.137.220

2.3 修改 Harbor 配置文件

Harbor 提供配置模板 harbor.yml.tmpl，需复制为正式配置文件并修改关键参数（如主机名、端口、证书路径、数据存储路径等）。

# 1. 复制配置模板为正式配置文件
cp /opt/harbor/harbor.yml.tmpl /opt/harbor/harbor.yml

# 2. 编辑配置文件
vi /opt/harbor/harbor.yml

修改以下关键配置项（其他默认配置可根据需求调整）：

# 1. 配置 Harbor 服务器的主机名（必须与证书中的 CN 或 SANs 一致）
hostname: 192.168.137.220

# 2. 关闭 HTTP（若需保留 HTTP，可跳过此步，但建议生产环境禁用）
# http:
#   port: 80

# 3. 启用 HTTPS 并配置证书路径（指向之前生成的服务器证书和私钥）
https:
  port: 443  # HTTPS 默认端口
  certificate: /data/ssl/server.crt  # 服务器证书路径
  private_key: /data/ssl/server.key  # 服务器私钥路径

# 4. 配置 Harbor 数据存储路径（默认 /data，可自定义）
data_volume: /data

# 5. 配置管理员初始密码（默认 Harbor12345，建议修改为复杂密码）
harbor_admin_password: Harbor12345

2.4 安装并启动 Harbor

# 1. 进入 Harbor 安装目录
cd /opt/harbor/

# 2. 运行准备脚本，生成 Docker Compose 配置文件（docker-compose.yml）
./prepare

# 3. 启动 Harbor 服务（-d 表示后台运行）
docker-compose up -d

# 4. 验证启动结果，若所有容器状态为 Up，则说明启动成功
docker-compose ps

安装成功后，终端会输出：✔ ----Harbor has been installed and started successfully.----

3. 访问 Harbor Web 界面

Harbor 提供 Web 管理界面，可通过浏览器进行仓库创建、用户管理、镜像上传 / 下载等操作。

3.1 登录 Web 界面

  1. 打开浏览器，输入 Harbor 地址：https://192.168.137.220（HTTPS 默认端口 443，无需额外指定）。
  2. 首次访问可能提示「证书不安全」（自签证书默认不被浏览器信任），选择「高级」→「继续访问」即可。
  3. 输入默认管理员账号和密码：
    • 用户名：admin
    • 密码：Harbor12345（若修改过配置文件，使用自定义密码）。

3.2 创建测试项目

登录后，点击左侧「项目」→「新建项目」，创建一个名为 test-images 的项目（用于后续测试镜像推送）：

  • 项目名称：test-images
  • 访问级别：选择「私有」（仅授权用户可访问）或「公开」（无需登录即可拉取镜像）。

4. 管理 Harbor 服务

日常运维中，可能需要停止、重启或查看 Harbor 服务状态，可通过以下命令操作：

# 1. 进入 Harbor 安装目录（必需，否则无法找到 docker-compose.yml 文件）
cd /opt/harbor/

# 2. 停止 Harbor 服务（-v 表示同时删除数据卷，谨慎使用！）
docker-compose down

# 3. 启动 Harbor 服务
docker-compose up -d

# 4. 重启 Harbor 服务（先停止再启动）
docker-compose down && docker-compose up -d

# 5. 查看 Harbor 服务日志（可指定容器名，如 nginx、postgres）
docker-compose logs -f  # 查看所有容器日志
docker-compose logs -f nginx  # 仅查看 Nginx 容器日志

5. 配置 Kubernetes 节点信任 Harbor

Kubernetes 节点（如 Worker 节点）通过 containerd 或 docker 拉取 Harbor 镜像时，需信任 Harbor 的 CA 根证书（否则会提示证书验证失败）。以下步骤以 containerd 为例（Kubernetes 1.24+ 默认容器运行时）。

5.1 在所有 Kubernetes 节点配置 CA 信任

# 1. 将 Harbor 的 CA 根证书复制到系统信任目录
sudo cp /data/ssl/ca.pem /etc/pki/ca-trust/source/anchors/harbor-ca.pem

# 2. 更新系统证书信任库
sudo update-ca-trust extract

# 3. 重启 containerd，使信任配置生效
sudo systemctl restart containerd

5.2 若使用 Docker 作为容器运行时，配置 Docker 信任 Harbor（可选）

若节点仍使用 Docker，需修改 Docker 配置文件，添加 Harbor 为「不安全仓库」（仅适用于 HTTP 或自签证书 HTTPS）：

# 1. 编辑 Docker 守护进程配置文件
sudo vi /etc/docker/daemon.json

# 2. 添加以下内容（IP 替换为 Harbor 服务器 IP）
{
  "insecure-registries": ["192.168.137.220"]
}

# 3. 重启 Docker 服务
sudo systemctl restart docker

6. 使用 Harbor 仓库

本节将演示如何通过 Docker 客户端登录 Harbor、推送镜像，以及如何在 Kubernetes 中拉取 Harbor 镜像。

6.1 Docker 客户端操作（推送镜像）

# 1. 登录 Harbor（输入管理员账号或项目授权用户账号）
docker login 192.168.137.220

# 2. 拉取一个测试镜像（以 busybox 为例）
docker pull busybox:1.29-4

# 3. 为镜像打标签（格式：Harbor地址/项目名/镜像名:标签）
docker tag busybox:1.29-4 192.168.137.220/test-images/busybox:1.29-4

# 4. 推送镜像到 Harbor 的 test-images 项目
docker push 192.168.137.220/test-images/busybox:1.29-4

# 5. （可选）登出 Harbor
docker logout 192.168.137.220

推送成功后，可在 Harbor Web 界面的「test-images」项目中看到该镜像。

6.2 Kubernetes 拉取 Harbor 镜像

若 Harbor 项目为「私有」，Kubernetes 需通过「镜像拉取密钥（imagePullSecrets）」登录 Harbor；若为「公开」，可直接拉取。

6.2.1 创建镜像拉取密钥

# 在 Kubernetes 集群中创建密钥（替换为实际的 Harbor 账号和密码）
kubectl create secret docker-registry harbor-secret \
  --docker-server=192.168.137.220 \
  --docker-username=admin \
  --docker-password=Harbor12345

6.2.2 部署 Pod 拉取 Harbor 镜像

创建 busybox-pod.yaml 文件，指定 Harbor 镜像和拉取密钥：

apiVersion: v1
kind: Pod
metadata:
  name: busybox
spec:
  containers:
  - name: busybox
    image: 192.168.137.220/test-images/busybox:1.29-4  # Harbor 镜像地址
    command: ["sh", "-c", "sleep 3600"]  # 让 Pod 持续运行
  imagePullSecrets:
  - name: harbor-secret  # 引用之前创建的镜像拉取密钥
  restartPolicy: Never

部署 Pod 并验证：

# 1. 创建 Pod
kubectl apply -f busybox-pod.yaml

# 2. 查看 Pod 状态，若 STATUS 为 Running 则说明拉取成功
kubectl get pods busybox

7. 总结

本教程从环境准备到实际使用，完整覆盖了 Harbor 私有镜像仓库的部署流程，核心要点包括：

  1. 基础依赖（Docker Compose）的安装；
  2. HTTPS 证书的生成与配置（解决证书验证失败问题）；
  3. Harbor 的安装、启动与 Web 管理；
  4. Kubernetes 节点的 CA 信任配置；
  5. 镜像的推送与 Kubernetes 拉取操作。

通过 Harbor，可实现私有镜像的统一管理，避免依赖公网仓库，同时提高镜像拉取速度和集群安全性。后续可根据需求扩展 Harbor 功能，如配置镜像同步、启用漏洞扫描等。
